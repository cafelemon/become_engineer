# 可评估的智能学习助手

## 在统一路线中的位置

- 什么时候开始：进入 LLM/Agent 后。
- 开始条件：P5.1 模型使用需要 Python 工程化与 Web/API；P5.2 以后再逐步加入检索、评估和数据库。
- NLP/Transformer 是模型方向深化，不是应用型 LLM 起步的强制前置。
- Tool Calling、工作流和 Agent 能力必须按里程碑顺序加入，不能跳过检索基线与固定评估集。

## 能力目标

使用本仓库的公开内容构建一个有来源、可评估、可调用受控工具并能持续演进的学习助手。

## 能力覆盖

- 工程基础：配置、测试、日志、安全、CI 和部署。
- Web 全栈：API、数据存储、认证和界面。
- LLM/Agent：结构化输出、RAG、Tool Calling、工作流和评估。
- CS 核心：检索、数据库、网络和系统设计在项目中持续回链。

## 前置课程

- Python 工程化。
- HTTP、API、数据库和基础 Web 交付能力。
- NLP/Transformer 与 LLM 使用基础。
- 基本安全边界、测试和可观测性。

## 工程要求

- 内容来源、索引版本和评估集可追溯。
- 模型调用与业务逻辑解耦。
- 工具使用白名单、参数校验和最小权限。
- 高风险动作要求人工确认。
- 日志、成本、延迟、错误和质量均可观察。
- 框架更换不得改变核心评估协议。

## 不纳入范围

- 不为 Dify、Coze、LangChain 或 LangGraph 分别建项目。
- 不在没有评估基线时直接做 Agent 或微调。
- 不使用本地 ignored 原素材作为公开知识库数据。

## 当前状态

P5.1 确定为 6 课，连续演进 v0.1–v0.6；默认使用标准库和确定性离线适配器，真实 provider 仅为显式可选验收，不要求或保存凭据。RAG、Tool Calling、Agent、微调、本地推理和多模态不进入本批。Tool Calling 知识内容和安全销售查询练习继续只作为 P5.5 的设计样品。
