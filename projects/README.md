# 项目实践

这里存放可公开的实践项目设计、代码和复盘。

项目应尽量覆盖真实工程流程：

- 需求定义
- 技术方案
- 开发实现
- 测试验证
- 部署运行
- 复盘改进

## 项目要求

- 标注所属主线和适合阶段。
- 一个项目应横跨多篇笔记和多个能力模块，通过里程碑逐步加码。
- 每个里程碑写清对应笔记、代码产出和验收标准。
- 在项目中持续链接回工程、语言、数学、CS 和 Web 基础。
- 尽量保留真实问题、取舍和复盘。
- 不提交私有数据、账号信息或受版权保护资料。
