# 项目

这里不是零散 demo 集合，而是把学习路线中的多个知识点整合到真实任务里的项目主干。

你不需要一开始就做完一个大项目。项目会随着课程推进逐步解锁：先完成前置课程，再进入对应里程碑；每个里程碑都会说明要产出什么、怎么验证、需要回到哪些基础知识。

## 项目怎么用

课程实践分成三级：

- 微练习：位于单节课程正文，验证一个知识点，不建立项目目录。
- 阶段作品：默认由连续3-6节课程形成；独立作品进入 `exercises/`，推进现有项目时成为项目里程碑。
- 长期项目：跨多门课程持续演进，用来整合工程、语言、CS、Web、AI 和 LLM/Agent 能力。

长期项目会放在本页展示。它们允许使用AI生成脚手架、测试和文档，但每个里程碑都必须保留人工审阅、主动修改、运行测试、排错和回归证据。具体要求见[课程内容规范](https://github.com/cafelemon/become_engineer/blob/main/docs/08_content_standard.md)。

## 学习中的项目关系

- 课程负责讲清概念和操作。
- 微练习负责验证单一知识点。
- 阶段作品负责整合连续课程能力。
- 长期项目负责把多个主线的知识合成完整工程能力。

因此，一个项目会横跨多篇课程；一篇课程也可能只推进项目中的一个小里程碑。
