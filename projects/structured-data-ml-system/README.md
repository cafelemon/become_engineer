# 结构化数据机器学习系统

## 在统一路线中的位置

- 解锁阶段：阶段5 AI基础。
- 解锁条件：完成Python工程化、必要数学、数据处理和机器学习评估前置。
- 项目服务化里程碑还要求阶段4的Web/API基础。

## 能力目标

围绕一个稳定公开数据集，完成数据检查、特征处理、训练验证、指标分析、模型比较、推理和基础服务化。

## 所属主线与阶段

- 编程语言：Python 数据处理。
- AI 基础：数学、机器学习和实验工程。
- 工程基础：配置、测试、日志和可重复运行。
- Web 全栈：最终阶段的简单推理 API。

## 前置课程

- Python、NumPy、pandas 和可视化。
- 线性代数、概率统计、损失函数和梯度的必要基础。
- Git、虚拟环境、测试和配置管理。

## 里程碑

| 阶段 | 项目能力 | 对应笔记计划 | 代码产出 | 验收 |
| --- | --- | --- | --- | --- |
| P2.1 | 数据契约与质量检查 | 数据集、缺失值与异常值 | 数据检查脚本 | 生成可重复质量报告 |
| P2.2 | EDA 与基线 | 描述统计、可视化与基线 | EDA 报告 | 明确目标、分布和基线指标 |
| P2.3 | 特征工程 | 编码、缩放与特征选择 | 特征流水线 | 训练和推理复用同一处理 |
| P2.4 | 训练与验证 | 划分、交叉验证和数据泄漏 | 训练脚本 | 验证集与测试集职责清楚 |
| P2.5 | 模型比较 | 线性模型、树模型与指标 | 比较报告 | 结论包含误差和成本 |
| P2.6 | 推理与复现 | 模型保存、配置和随机种子 | 推理入口 | 新环境可重复推理 |
| P2.7 | 工程化 | 测试、日志和简单 API | 服务入口 | 输入校验和核心测试通过 |

## 工程要求

- 原始数据、处理数据和模型产物分层。
- 训练配置、数据版本和指标可追溯。
- 防止训练测试污染和目标泄漏。
- API 只是交付阶段，不替代机器学习学习目标。

## 不纳入范围

- 不引入深度学习来追求无意义的复杂度。
- 不延伸到 Agent。
- 不把单一最高指标作为唯一成功标准。

## 当前状态

规划完成，数据集在正式实施前根据许可、规模和任务清晰度确定。
