# Python 内容分析工具

## 在统一路线中的位置

- 解锁阶段：阶段1 Python起步完成后。
- 首个入口：P1.1目录扫描与Markdown/JSON读取。
- 后续里程碑随阶段2的Python工程化能力逐步解锁，不要求起步时一次完成。

## 能力目标

构建一个能够扫描 Markdown、JSON 和目录结构，生成内容清单、重复候选、风险标签和覆盖报告的命令行工具。

## 所属主线与阶段

- 工程基础：命令行、目录、日志、测试、文档和 CI。
- 编程语言：Python 基础与工程化。
- AI 基础前置：数据处理和可重复实验习惯。

## 前置课程

- Git、Shell、路径和环境变量。
- Python 语法、数据结构、函数、文件和模块。
- 异常处理、日志和单元测试。

## 工程要求

- 核心逻辑与 CLI 分离。
- 输入只读，输出写入指定目录。
- 分类规则可配置并记录版本。
- 日志不包含原文、凭据或私人路径。
- 测试使用小型固定样本，不提交版权资料。

## 不纳入范围

- 不在首版引入向量数据库或 LLM 自动分类。
- 不将当前外部素材原文打包进项目。
- 不追求一次解决全部语义重复。
