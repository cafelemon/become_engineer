# Python 内容分析工具

## 在统一路线中的位置

- 什么时候开始：完成 Python 起步后；当前已满足 P1.1 前置，可以与双主修并行启动。
- 首个入口：P1.1目录扫描与Markdown/JSON读取。
- 后续里程碑随Python 工程化能力逐步解锁，不要求起步时一次完成。
- 进入项目前，需要先完成变量、条件、循环、函数、数据结构、文件读取、模块、基本调试和最小测试，并验收学习进度报告器阶段作品。

## 能力目标

构建一个能够扫描 Markdown、JSON 和目录结构，生成内容清单、重复候选、风险标签和覆盖报告的命令行工具。

## 能力覆盖

- 工程基础：命令行、目录、日志、测试、文档和 CI。
- 编程语言：Python 基础与工程化。
- AI 基础前置：数据处理和可重复实验习惯。

## 前置课程

- Git、Shell、路径和环境变量。
- Python 语法、数据结构、函数、文件和模块。
- 异常处理和单元测试；日志在后续工程化里程碑补充。

## 工程要求

- 核心逻辑与 CLI 分离。
- 输入只读，输出写入指定目录。
- 分类规则可配置并记录版本。
- 日志不包含原文、凭据或私人路径。
- 测试使用小型固定样本，不提交版权资料。

## 不纳入范围

- 不在首版引入向量数据库或 LLM 自动分类。
- 不将当前外部素材原文打包进项目。
- 不追求一次解决全部语义重复。
