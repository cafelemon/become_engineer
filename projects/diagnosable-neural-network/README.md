# 可诊断神经网络训练系统

## 在统一路线中的位置

- 什么时候开始：完成机器学习后进入深度学习。
- 开始条件：能够区分训练、验证和最终测试，理解损失、过拟合与可复现实验。
- 本项目先建立通用训练系统；卷积网络、序列模型和 Transformer 留给视觉与 NLP 专项。

## 能力目标

围绕一份固定种子生成的二维二分类教学数据，从张量契约推进到前向网络、自动微分、训练循环、验证与正则化、梯度诊断、检查点恢复和离线推理交付。最终不仅要得到一个模型，还要能解释“数据如何流动、梯度为何成立、训练何时失败、产物怎样恢复”。

## 能力覆盖

- 深度学习基础：张量、模块、激活、损失、反向传播和优化器。
- 实验判断：训练/验证职责、学习曲线、过拟合、正则化与对照实验。
- 工程能力：固定配置、结构化历史、失败诊断、检查点、恢复和推理契约。
- 系统边界：CPU 低成本路径、设备迁移、确定性限制和可信产物加载。

## 前置课程

- 数学与数据：向量矩阵、导数、概率、划分、指标和可复现实验。
- 机器学习：基线、交叉验证、正则化、误差分析和交付边界。
- Python 工程：模块、类型、异常、测试、文件与资源生命周期。

## 工程约束

- Python 3.12 与 PyTorch 2.13.0，默认 CPU；不要求 GPU、云服务或在线数据。
- 数据由固定种子在本机生成，不包含个人数据，也不代表真实业务分布。
- 固定输出不记录耗时、硬件型号或动态设备可用性。
- 每一版保留真实 unittest、确定性报告和至少一个可解释失败路径。
- 只有本地生成且经过审查的检查点可以加载；校验和不能把不可信文件变成可信文件。

## 不纳入范围

- 不在通用基础课中提前教授 CNN、RNN、Attention 或 Transformer。
- 不追求大模型、分布式训练、混合精度、GPU 性能或云端部署。
- 不把一次小数据高准确率当作真实场景效果，也不把固定随机种子误写为跨平台绝对复现。
