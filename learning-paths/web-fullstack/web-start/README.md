# Web 起步

Web 起步把学习进度报告器从终端带进浏览器。这里不急着上框架，也不要求你先会数据库：先看懂页面，再让页面动起来，最后才把它接到 Python API。

| --- | --- | --- | --- |
| [浏览器、HTML 与第一张学习卡片](01-browser-html-learning-card.md) | Web v0.1 | 有语义结构、可以直接打开的学习卡片 | 已开放 |
| [CSS 布局、视觉层级与手机适配](02-css-layout-responsive-dashboard.md) | Web v0.2 | 桌面与手机都能读的页面 | 已开放 |

| [HTTP、JSON 与本地 API](04-http-json-local-api.md) | Web v0.4 | 从 FastAPI 读取真实学习报告 | 已开放 |

四节已经全部开放。建议第一次按顺序做；已有基础者可以用每节开头的跳过检查只补缺口。

## 四类学习者怎么走

- **小白兴趣／小白求职**：四节按顺序完成。求职画像额外保留结构说明、错误截图和测试结果。
- **有基础兴趣／有基础求职**：先做每节开头的跳过检查。能独立复现且解释关键选择，就只补自己的缺口。
- **兴趣画像**：主课和项目都要做，不默认加入面试背诵或限时题。
- **求职画像**：使用同一份项目，在课程标出的“求职加练”处补充代码解释、测试与故障复盘。

## 完成 Web 起步以后

你应该能独立写一张页面、适配窄屏、处理一次用户操作，并把浏览器请求接到本地 Python 服务。Web 核心的前置已经满足；下一阶段会学习 TypeScript、组件边界、FastAPI 契约和数据库。
