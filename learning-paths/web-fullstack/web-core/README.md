## 四类学习者怎么走

- **小白兴趣**：先完成 Web 起步，再按顺序学习。每节先跑通，再改一个字段，不额外加入机考或面试题。
- **有基础兴趣**：先做课程开头的跳过检查。契约、持久化或表单哪一层解释不清，就补哪一节。
- **小白求职**：主课照常学习，每节再保存一次故障记录和测试输出，逐渐形成项目讲解材料。
- **有基础求职**：可以快速通过基础复现，但必须完成契约漂移、错误处理和设计取舍的求职加练。

兴趣与求职使用同一份项目代码。区别在于求职画像会多做故障复盘、接口评审和项目表达，不会复制一套内容。

## 与 CS 系统基础的关系

第一次学习 Web Core 不要求先学完完整操作系统、网络和数据库原理。课程会在需要时引入最少的 HTTP、进程和存储知识，并链接到对应的 CS 深化入口。

进入认证、并发、部署、性能和可观测性之前，再补齐进程、内存、网络协议、事务与索引等系统基础。这样新手不会被前置知识堵住，有基础者也不会停在“框架会用但解释不清”的位置。

## 这一组的完成标准

- 页面能拒绝不符合契约的运行时数据，而不是只依赖编辑器提示。
- 学习记录可以经过模型、迁移和事务保存到本地数据库。
- API 能完成查询、创建、更新、删除与分页，并说明幂等边界。
- 表单能处理客户端校验、服务端校验、提交中、成功与失败状态。
- 项目保持可重复启动、可测试、手机可读，并留下真实的故障修复记录。
