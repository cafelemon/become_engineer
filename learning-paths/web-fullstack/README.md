# Web 全栈

## 定位

Web 全栈线用于把前端、后端、数据库、API、部署和运维入口串成完整产品能力。目标是能从需求出发做出可访问、可维护、可验证的应用。

本主线在CS最小核心完成后进入，并与CS深化并行。

数据库实践默认以 PostgreSQL 和 `psql` 为主，因为它在后端、数据、AI应用和部署场景中覆盖面更广；MySQL作为常见生产数据库做对照补充。SQL 是共同语言，但路线不能只停留在抽象 SQL。

## 适合人群

- 已经具备一门编程语言基础，想做真实应用的人。
- 想理解前后端如何协作的人。
- 想为 AI/Agent 应用补齐产品交付能力的人。

## 阶段划分

1. 浏览器与前端基础：HTML、CSS、JavaScript。
2. TypeScript与组件化：类型、状态、表单和请求。
3. Python后端：FastAPI、路由、数据校验、错误处理和测试。
4. 数据库与API：PostgreSQL、psql、SQL、建模、迁移、REST、OpenAPI、认证和权限，并补充MySQL差异。
5. 完整应用：前后端协作、配置、日志和安全。
6. 部署运维：监控、发布、回滚和性能分析。
7. Java补充分支：Java后端与Spring生态，可与Python后端对照。

## 推荐学习顺序

1. 先完成静态页面和JavaScript交互，再进入TypeScript和组件化。
2. 使用Python/FastAPI建立API和测试。
3. 增加PostgreSQL数据库、迁移、登录、权限和完整数据流程，按需要对照MySQL。
4. 同期回链CS中的HTTP、网络、数据库、并发和系统设计。
5. 最后完成部署、监控和回滚。

## 进入与离开条件

- 进入条件：CS最小核心验收完成。
- AI应用交付解锁：能够完成含前端、API、数据库、认证和测试的应用。
- LLM/Agent解锁：同时具备API、数据库、权限、日志和部署基础。

详细顺序见[小白统一学习路线](../beginner-roadmap.md)。
