# Web 全栈

## 定位

Web 全栈线用于把前端、后端、数据库、API、部署和运维入口串成完整产品能力。目标是能从需求出发做出可访问、可维护、可验证的应用。

## 适合人群

- 已经具备一门编程语言基础，想做真实应用的人。
- 想理解前后端如何协作的人。
- 想为 AI/Agent 应用补齐产品交付能力的人。

## 阶段划分

1. 前端基础：HTML、CSS、JavaScript、TypeScript、组件化。
2. 后端基础：HTTP、路由、认证、业务逻辑、错误处理。
3. 数据库：SQL、建模、索引、迁移、备份。
4. API 与集成：REST、OpenAPI、鉴权、第三方服务。
5. 部署与运维：环境变量、日志、监控、发布、回滚。

## 推荐学习顺序

1. 先做一个静态页面，再做可交互前端。
2. 增加后端 API 和数据库。
3. 完成登录、权限、数据增删改查和部署。
4. 用真实项目复盘性能、错误处理和可维护性。
