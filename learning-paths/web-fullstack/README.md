# Web 全栈

## 定位

Web 全栈用于把浏览器、前端、后端、数据库、API、安全和部署串成完整产品能力。目标不是做出一个只能本机演示的页面，而是能从需求出发构建可访问、可维护、可测试、可发布的应用。

数据库实践默认以PostgreSQL和`psql`为主，MySQL作为常见生产数据库对照。SQL是共同语言，但课程不能停留在语法练习。

## 在默认路线中的位置

- 进入条件：CS必修主干验收完成。
- 必修主干完成后：具备交付普通Web应用的完整闭环，并满足AI应用服务化和LLM/Agent的Web前置。
- 推荐深化和专业选修：不阻塞AI路线，但可以成为实时系统、生产后端和SRE等方向的前置。
- 完整关系见[完整课程地图](../curriculum-map.md)。

## 适合人群

- 想构建真实前后端应用的人。
- 想理解浏览器、API和数据库如何协作的人。
- 想为AI或Agent应用补齐产品交付能力的人。
- 想继续进入前端、后端、实时通信或DevOps/SRE方向的人。

## 必修主干

| 模块 | 知识范围 | 实践与验收 |
| --- | --- | --- |
| 浏览器与页面 | 浏览器基本模型、HTML、CSS、语义结构、布局和基础可访问性 | 完成可在桌面与移动视口使用的页面并检查结构 |
| JavaScript与TypeScript | 语法、对象、函数、异步、模块、类型和错误处理 | 完成带异步数据请求和错误状态的交互功能 |
| 组件化前端 | 组件、状态、表单、路由、请求、加载与失败状态 | 完成一个可维护的前端功能流并通过基本测试 |
| Python后端 | FastAPI、路由、请求响应模型、校验、异常、依赖和测试 | 完成具有清晰契约和错误响应的API |
| 数据库与数据访问 | PostgreSQL、psql、关系建模、SQL、迁移、事务和MySQL差异 | 完成数据库建模、迁移、增删改查和回滚验证 |
| API与权限 | HTTP、REST、OpenAPI、认证、授权、输入验证和最小权限 | 完成登录、身份识别和至少两种权限场景测试 |
| 完整应用 | 前后端协作、配置、日志、自动化测试和安全边界 | 交付可重复启动、可测试、可观察的完整应用 |
| 部署闭环 | Docker、环境变量、发布、监控、备份和回滚 | 在隔离环境完成部署、健康检查和一次回滚演练 |

### 必修完成标准

- 能独立完成包含前端、API、PostgreSQL、认证和测试的数据流程。
- 能根据OpenAPI或等价契约解释前后端输入、输出和错误。
- 能使用迁移维护数据库结构，而不是手工修改生产表。
- 能区分配置、密钥和普通数据，并避免把敏感信息提交到仓库。
- 能部署应用、检查健康状态，并说明失败时如何回滚。

## 推荐学习顺序

1. 完成页面、JavaScript/TypeScript和组件化前端。
2. 使用Python/FastAPI建立有测试的API。
3. 增加PostgreSQL、迁移、认证、权限和完整数据流。
4. 完成部署、健康检查、监控和回滚闭环。
5. 根据目标选择推荐深化；满足前置后再进入专业选修。
