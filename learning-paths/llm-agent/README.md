# LLM/Agent

## 定位

LLM/Agent用于学习大模型应用与智能体系统，从可验证的模型调用推进到检索、评估、工具、工作流、安全和真实交付。目标不是堆框架或让Agent无限自主，而是建立可测量、可观察、可控制的系统。

本主线不能用“会调用模型API”代替Python工程、Web/API、NLP/Transformer、实验评估和安全前置。

## 在默认路线中的位置

- 进入条件：Python工程化、Web必修主干、AI必修主干中的NLP/Transformer与评估完成。
- 必修主干完成后：具备交付有引用、可评估、可使用工具的有界LLM应用能力。
- 推荐深化和专业选修：不阻塞默认路线结束，但本地推理、微调、多模态等方向具有跨主线前置。
- 完整关系见[完整课程地图](../curriculum-map.md)。

## 适合人群

- 想构建可验证LLM应用的人。
- 想理解RAG、Tool Calling和Agent工作流的人。
- 想把模型能力接入真实数据、API和业务流程的人。
- 想继续进入微调、本地推理、多模态或专业Agent方向的人。

## 必修主干

| 模块 | 知识范围 | 实践与验收 |
| --- | --- | --- |
| 模型使用基础 | Token、上下文、模型能力边界、Prompt、结构化输出和失败类型 | 完成固定输入集的结构化输出实验并记录失败 |
| 检索基线 | 文档处理、关键词检索、分块、召回、排序和引用 | 先建立不用LLM也能检查的检索基线 |
| RAG与引用 | Embedding、向量检索、重排、上下文组装、回答和引用 | 完成带可追溯引用的RAG，并验证引用与原文一致 |
| 评估 | 固定评估集、检索指标、答案质量、忠实度、人工复核和回归 | 在增加复杂能力前建立可重复评估，并保留基线 |
| Tool Calling | 工具Schema、参数验证、执行边界、权限、结果回传和错误处理 | 完成正常、非法参数、未知工具、失败和恶意输入测试 |
| 有界工作流 | 状态、步骤、分支、重试、人工确认、失败恢复和终止条件 | 完成可观察、可停止、可恢复的工作流 |
| 工程与安全 | 日志、成本、延迟、密钥、提示注入、最小权限、部署和回滚 | 完成风险清单、监控指标和一次失败恢复或回滚演练 |

### 必修完成标准

- 能为同一任务保存检索或非LLM基线，并用固定评估集比较改动。
- RAG答案包含可核查引用，无法回答时能明确失败而不是编造。
- 模型只提出工具调用，应用负责参数验证、权限判断和实际执行。
- Agent工作流具有状态、终止条件、错误恢复和需要时的人工确认。
- 能记录质量、成本、延迟和安全风险，并说明部署与回滚方式。

## 推荐学习顺序

1. 先完成结构化输出和检索基线。
2. 建立带引用的RAG，并立即加入固定评估集。
3. 加入Tool Calling、参数校验和权限边界。
4. 再进入有界工作流、状态、失败恢复和人工确认。
5. 完成工程、安全和部署闭环后，按目标选择推荐深化或专业选修。
