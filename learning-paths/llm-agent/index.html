    LLM/Agent¶

    定位¶

    LLM/Agent用于学习大模型应用与智能体系统，从可验证的模型调用推进到检索、评估、工具、工作流、安全和真实交付。目标不是堆框架或让Agent无限自主，而是建立可测量、可观察、可控制的系统。

    本主线不能用“会调用模型API”代替Python工程、Web/API、NLP/Transformer、实验评估和安全前置。

    在默认路线中的位置¶

    • 进入条件：Python工程化、Web必修主干、AI必修主干中的NLP/Transformer与评估完成。
    • 必修主干完成后：具备交付有引用、可评估、可使用工具的有界LLM应用能力。
    • 推荐深化和专业选修：不阻塞默认路线结束，但本地推理、微调、多模态等方向具有跨主线前置。
    • 完整关系见完整课程地图。

    适合人群¶

    • 想构建可验证LLM应用的人。
    • 想理解RAG、Tool Calling和Agent工作流的人。
    • 想把模型能力接入真实数据、API和业务流程的人。
    • 想继续进入微调、本地推理、多模态或专业Agent方向的人。

    必修主干¶

    模块 知识范围 实践与验收
    模型使用基础 Token、上下文、模型能力边界、Prompt、结构化输出和失败类型 完成固定输入集的结构化输出实验并记录失败
    检索基线 文档处理、关键词检索、分块、召回、排序和引用 先建立不用LLM也能检查的检索基线
    RAG与引用 Embedding、向量检索、重排、上下文组装、回答和引用 完成带可追溯引用的RAG，并验证引用与原文一致
    评估 固定评估集、检索指标、答案质量、忠实度、人工复核和回归 在增加复杂能力前建立可重复评估，并保留基线
    Tool Calling 工具Schema、参数验证、执行边界、权限、结果回传和错误处理 完成正常、非法参数、未知工具、失败和恶意输入测试
    有界工作流 状态、步骤、分支、重试、人工确认、失败恢复和终止条件 完成可观察、可停止、可恢复的工作流
    工程与安全 日志、成本、延迟、密钥、提示注入、最小权限、部署和回滚 完成风险清单、监控指标和一次失败恢复或回滚演练

    必修完成标准¶

    • 能为同一任务保存检索或非LLM基线，并用固定评估集比较改动。
    • RAG答案包含可核查引用，无法回答时能明确失败而不是编造。
    • 模型只提出工具调用，应用负责参数验证、权限判断和实际执行。
    • Agent工作流具有状态、终止条件、错误恢复和需要时的人工确认。
    • 能记录质量、成本、延迟和安全风险，并说明部署与回滚方式。

    推荐深化¶

    模块 用途与适合人群 前置与推荐时机 知识与实践 客观验收 跳过影响与关联
    检索工程 面向复杂知识库、搜索和高质量RAG 检索基线、RAG与引用必修完成后 分块策略、混合检索、重排、查询改写、过滤、召回诊断和索引更新 使用固定数据集比较检索策略并解释错误 不影响简单RAG；跳过会限制Text2SQL外知识检索和复杂知识助手；关联智能学习助手，素材池已审计
    评估系统 面向持续迭代、团队协作和高风险应用 固定评估集和一个完整LLM功能完成后 数据集版本、自动评估、人工量表、回归、在线反馈和评估偏差 建立可重复评估流水线并捕获一次回归 不影响实验Demo；跳过会限制生产Agent和微调决策；关联智能学习助手，素材池已审计
    Agent状态与记忆 面向多步骤任务、长期状态和协作流程 Tool Calling、有界工作流和数据库基础后 状态模型、短期/长期记忆、计划、检查点、恢复和人工确认 完成可中断恢复的多步骤任务并验证状态一致性 不影响单轮应用；跳过会限制复杂Coding/Research Agent；关联智能学习助手，Agent素材池已审计
    上下文工程 面向长文档、多来源和复杂任务 RAG、评估和NLP/Transformer基础后 上下文选择、压缩、缓存、历史管理、指令优先级和信息丢失分析 在固定预算下比较质量、成本和上下文策略 不影响短任务；跳过会限制长上下文和多Agent系统；关联智能学习助手，LLM素材池已审计
    成本、延迟、可观测性与安全 面向真实交付和生产运行 完成一个可部署LLM应用后 Token与调用成本、缓存、并发、追踪、提示注入、数据边界、告警和回滚 完成压测、安全测试、成本报告和故障恢复 不影响本地实验；跳过会限制生产交付和高风险场景；关联智能学习助手，部署素材已审计

    专业选修¶

    模块 用途与适合人群 前置与推荐时机 知识与实践 客观验收 跳过影响与关联
    微调与对齐 面向评估证明Prompt/RAG不足的稳定任务 AI深度网络、数据治理、评估系统完成后 数据构建、监督微调、偏好与对齐认知、评估和回滚 与基线比较，证明收益、成本和失败风险 不影响大多数LLM应用；跳过仍可使用Prompt/RAG；进入专项实验，素材池已审计
    本地推理与量化 面向隐私、离线、成本和性能方向 C++系统深化、深度网络与Transformer、性能分析完成后 模型格式、推理运行时、量化、显存/内存、吞吐延迟和部署 完成不同配置的质量、资源和性能比较 不影响云API应用；跳过则不进入本地模型和llama.cpp性能方向；已登记项目候选，部署素材已审计
    Text2SQL 面向结构化数据查询和分析助手 数据库工程、API安全、评估系统和Tool Calling完成后 Schema理解、查询生成、权限、只读执行、验证和结果解释 在受限数据库上通过正确性、安全和拒绝测试 不影响普通RAG；跳过则不进入数据库问答；素材已登记待筛选
    多模态Agent 面向文档、图像、音频或视频工作流 AI多模态选修、评估系统和有界工作流完成后 多模态输入、工具、状态、引用和质量安全评估 完成一个有固定评估集的多模态工作流 不影响文本Agent；跳过则不进入多模态系统；进入项目候选，素材池已审计
    Coding/Research Agent 面向代码、检索、分析和长任务自动化 Agent状态与记忆、上下文工程、评估系统和安全深化后 计划、工具、沙箱、证据、检查点、人工审阅和失败恢复 在固定任务集上完成成功率、安全和人工复核评估 不影响普通助手；跳过则不进入长任务Agent方向；进入项目候选，Agent素材池已审计

    项目主干¶

    • 可评估的智能学习助手：承载检索、引用、RAG、评估、Tool Calling、工作流、可观测性和部署。

    当前已完成的Tool Calling知识单元：

    • 笔记：Tool Calling：让模型安全地使用外部能力
    • 练习：安全销售查询 Tool Calling
    • 对应项目里程碑：P5.5 Tool Calling

    推荐学习顺序¶

    1. 先完成结构化输出和检索基线。
    2. 建立带引用的RAG，并立即加入固定评估集。
    3. 加入Tool Calling、参数校验和权限边界。
    4. 再进入有界工作流、状态、失败恢复和人工确认。
    5. 完成工程、安全和部署闭环后，按目标选择推荐深化或专业选修。
