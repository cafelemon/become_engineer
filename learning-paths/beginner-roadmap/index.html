    小白统一学习路线¶

    这是 Become Engineer 的唯一学习顺序基准。六条主线仍然用于组织内容，但新学习者应按本路线逐阶段推进。

    总顺序¶

    阶段 0 工程基础前置
  -> 阶段 1 Python 起步
    -> 阶段 2 Python / C++ 双主修
      -> 阶段 3 CS 最小核心
        -> 阶段 4 Web 实践 / CS 深化并行
          -> 阶段 5 AI 基础
            -> 阶段 6 LLM / Agent

    可选补充：

    • Java：在阶段 2 之后开始，作为完整的主流语言与后端生态补充路线。
    • 强化学习：在机器学习和深度学习基础完成后进入，不是 LLM/Agent 前置。
    • 多模态、Text2SQL、视觉、金融 AI、本地推理：完成对应前置后再选择。

    算法口径：

    • CS 阶段的算法指通用算法基础：数据结构、复杂度、递归、排序、查找、动态规划和图等。
    • AI 阶段的算法指模型算法：机器学习、深度学习、NLP/Transformer、强化学习和多模态等。
    • 两者都重要，但学习顺序不同；不能用 AI 模型算法替代 CS 通用算法，也不能把刷题当成机器学习。

    使用规则¶

    1. 当前阶段的必学内容和验收未完成，不进入下一阶段主线。
    2. 标记为“并行”的内容可以交替推进，不要求同一天同时学习。
    3. 项目只在达到解锁条件后进入，不能替代前置课程。
    4. 项目中遇到基础知识时，必须链接回对应课程并再次复习。
    5. 外部资料只是候选来源，不代表对应课程已经完成建设。

    阶段 0：工程基础前置¶

    学习目标¶

    能够在本地创建、运行、修改和验证一个学习仓库，知道文件在哪里、命令做了什么、错误如何记录。

    前置条件¶

    无。

    必学单元¶

    1. 学习方法：目标拆解、主动练习、记录问题、阶段复盘。
    2. 文件系统：目录、路径、扩展名、相对路径和绝对路径。
    3. 终端与 Shell：进入目录、查看文件、运行命令、读取退出状态。
    4. 编辑器：打开项目、搜索、修改文件、使用终端。
    5. Markdown：标题、列表、链接、代码块和文档结构。
    6. Git：仓库、状态、暂存、提交、历史和远程仓库。
    7. 开发环境：解释器、编译器、环境变量和依赖的基本概念。
    8. Docker最小认知：镜像、容器、端口、挂载和环境变量的基本含义。
    9. 验证习惯：运行、观察输出、复现错误、记录命令和结果。

    暂不要求¶

    • 不要求先学完整单元测试、CI、部署或系统设计。
    • 不要求先写复杂 Dockerfile、docker compose 或生产部署脚本。
    • 这些工程能力在有代码和项目后持续深化。

    阶段验收¶

    • 能创建一个 Git 仓库并完成至少 3 次有意义的提交。
    • 能从终端进入项目目录并运行一个给定命令。
    • 能写一份包含目标、步骤、结果和问题的 Markdown 学习记录。
    • 能解释 Docker 解决的环境一致性问题，并知道 docker --version、docker run、docker ps、docker logs 和 docker stop 的用途。
    • 能根据错误信息指出失败发生在哪条命令，而不是只说“不能运行”。

    下一步¶

    按工程基础的前置课程完成上述验收后，进入阶段1 Python起步。

    阶段 1：Python 起步¶

    学习目标¶

    使用 Python 理解最基本的程序结构，并具备继续学习第二门语言的共同概念基础。

    前置条件¶

    • 阶段 0 验收完成。

    必学单元¶

    1. 变量、基本类型、输入和输出。
    2. 条件、循环和布尔逻辑。
    3. 函数、参数、返回值和作用域。
    4. 字符串、列表、字典、集合和元组。
    5. 文件读取、JSON 和简单目录操作。
    6. 模块、导入和虚拟环境的基本使用。
    7. 异常信息、基本调试和最小测试意识。

    项目解锁¶

    完成本阶段后，可以进入 Python 内容分析工具 的 P1.1。

    阶段验收¶

    • 能独立编写一个读取 JSON 或文本文件并输出统计结果的脚本。
    • 能把重复代码提取成函数。
    • 能解释列表和字典分别适合什么场景。
    • 能创建虚拟环境并运行项目命令。
    • 能根据 traceback 定位到具体文件和行。

    下一步¶

    进入阶段 2，启动 Python 与 C++ 双主修。

    阶段 2：Python / C++ 双主修¶

    学习目标¶

    用 Python 建立快速实现和工程化能力，同时用 C++ 建立类型、编译、内存和系统意识。

    前置条件¶

    • 阶段 1 验收完成。

    Python 主线¶

    1. 类型提示、模块组织和包结构。
    2. 命令行参数、配置、日志和异常设计。
    3. 单元测试、测试数据和可重复运行。
    4. NumPy、pandas 和基础数据处理。
    5. API 与自动化所需的标准库能力。

    C++ 主线¶

    1. 编译、链接、头文件和源文件。
    2. 类型、引用、指针、const 和作用域。
    3. 函数、类、对象生命周期和 RAII。
    4. STL 容器、迭代器和算法。
    5. 内存、智能指针、错误处理和调试。
    6. CMake、测试和基础工程组织。

    并行方式¶

    • Python 保持每周持续编码，用于工具、数据和自动化。
    • C++ 从共同概念重新实现小练习，重点理解与 Python 不同的编译、类型和资源管理。
    • 不要求两门语言章节编号完全同步。

    Java 补充路线¶

    Java不是默认必修，但提供完整学习入口：

    1. 语法、类型和控制流。
    2. 面向对象、异常和集合。
    3. 泛型、IO、并发基础。
    4. JVM、构建工具和测试。
    5. Spring 等后端生态在 Web 阶段进入。

    阶段验收¶

    • Python：能够完成带 CLI、日志和测试的小型工具。
    • C++：能够用 CMake 构建一个多文件程序，并正确使用 STL 和 RAII。
    • 能比较 Python 与 C++ 在类型、执行方式、内存和错误处理上的差异。

    下一步¶

    进入阶段 3 CS 最小核心。Python 与 C++ 继续在练习和项目中并行深化。

    阶段 3：CS 最小核心¶

    学习目标¶

    在开始完整 Web 应用前，掌握解释程序性能、网络请求和数据存储所需的最小 CS 基础。

    前置条件¶

    • 阶段 2 的 Python 和 C++ 基础验收完成。

    必学单元¶

    1. 数据结构：数组、链表、栈、队列、哈希表、树。
    2. 通用算法与复杂度：查找、排序、递归、动态规划、图的基本概念、时间和空间复杂度。
    3. 计算机运行基础：进程、线程、内存、文件和系统调用概念。
    4. 网络最小集：客户端/服务端、IP、TCP、DNS、HTTP。
    5. 数据库最小集：关系模型、表、主键、SQL、索引和事务概念；实践工具以 PostgreSQL 和 psql 为主，MySQL 作为常见对照。
    6. 数据交换：文本、JSON、编码和序列化。

    练习方式¶

    • 算法和数据结构优先使用 C++ 与 Python 对照实现。
    • HTTP 和数据库通过小型命令行实验验证；数据库默认先用 PostgreSQL/psql，SQLite 只用于轻量本地练习，MySQL 在 Web 阶段做差异对照。
    • 不要求先学完整操作系统、网络协议栈或数据库实现。

    阶段验收¶

    • 能为常见容器操作给出基本复杂度判断。
    • 能描述浏览器或客户端请求 API 的基本过程。
    • 能设计一个简单关系表并完成增删改查。
    • 能解释进程与线程、内存与磁盘的基本区别。

    下一步¶

    进入阶段 4：Web 实践与 CS 深化并行。

    阶段 4：Web 实践 / CS 深化并行¶

    学习目标¶

    做出可访问的完整应用，同时用 CS 原理解释和改进真实工程问题。

    前置条件¶

    • 阶段 3 验收完成。

    Web 主线¶

    1. HTML、CSS 和浏览器基础。
    2. JavaScript，再进入 TypeScript。
    3. 前端组件、状态、表单和请求。
    4. Python 后端、FastAPI、数据校验和错误处理。
    5. REST、OpenAPI、认证和权限。
    6. PostgreSQL、psql、SQL、迁移、备份和数据访问层；根据项目需要补充 MySQL 差异。
    7. 测试、日志、部署、监控和回滚。

    CS 深化主线¶

    1. 操作系统：进程线程、同步、内存和文件系统。
    2. 网络：TCP、HTTP、连接、缓存和代理。
    3. 数据库：PostgreSQL优先的事务、索引、查询优化和并发；理解 MySQL 在类型、索引、事务和部署上的常见差异。
    4. 系统设计：缓存、队列、可用性、一致性和扩展。
    5. 安全：认证、授权、输入校验、密钥和最小权限。

    Java 补充¶

    选择 Java 路线的学习者可在本阶段加入 Java 后端、构建工具、测试和 Spring 生态，并与 Python 后端做对照。

    阶段验收¶

    • 能完成一个包含前端、API、数据库、认证和测试的应用。
    • 能解释一条请求从浏览器到数据库再返回的过程。
    • 能定位常见前端、后端、网络和数据库问题。
    • 能写出部署、配置、日志和回滚说明。

    下一步¶

    • 希望继续产品和系统方向：继续深化 Web 与 CS。
    • 希望进入 AI：进入阶段 5。

    阶段 5：AI 基础¶

    学习目标¶

    理解数据、模型、训练和评估的完整闭环，并能够通过实验判断模型是否有效。

    本阶段的“算法”指模型算法，不替代 CS 阶段的通用算法。机器学习、深度学习、NLP/Transformer 和强化学习都归入 AI 基础，其中 NLP/Transformer 是进入 LLM/Agent 的关键桥梁，强化学习是独立进阶分支。

    前置条件¶

    • Python 工程化和数据处理。
    • 数据结构、复杂度和基本实验方法。

    必学顺序¶

    1. AI数学前置：线性代数、概率统计、导数、梯度与优化直觉。
    2. 数据清洗、EDA、特征、数据划分和评估指标。
    3. 监督学习、无监督学习和可靠验证。
    4. 结构化数据机器学习系统。
    5. 神经网络、反向传播和 PyTorch。
    6. NLP、序列模型、Attention 和 Transformer。
    7. 文本分类模型演进。

    独立并行分支¶

    • 强化学习控制实验：完成机器学习和深度学习前置后进入。

    阶段验收¶

    • 能说明训练集、验证集和测试集的职责。
    • 能识别数据泄漏、过拟合和错误指标。
    • 能复现一个传统 ML 模型和一个深度学习模型。
    • 能提交包含配置、指标、误差分析和复现步骤的报告。

    下一步¶

    进入阶段 6 LLM/Agent，或继续强化学习、多模态等 AI 分支。

    阶段 6：LLM / Agent¶

    学习目标¶

    构建有来源、可评估、可调用受控工具并能工程交付的 LLM 应用。

    前置条件¶

    • Python 工程化。
    • Web/API、数据库、认证和测试。
    • NLP/Transformer 基础。
    • 机器学习阶段形成的实验评估和安全意识。

    必学顺序¶

    1. 模型能力边界、Prompt、上下文和结构化输出。
    2. 关键词检索基线、引用和错误分析。
    3. 文档加载、分块、嵌入、向量检索和 RAG。
    4. 固定评估集、检索质量、忠实度和答案质量。
    5. Tool Calling、权限、参数校验和错误恢复。
    6. 状态、工作流、规划、记忆和人工确认。
    7. 日志、成本、延迟、安全、部署和回滚。
    8. 可评估的智能学习助手。

    智能学习助手从完成关键词检索基线、引用和固定评估集后逐步解锁；不能在只有Prompt知识时直接开始Agent里程碑。

    阶段验收¶

    • 回答能够追溯到明确来源。
    • 每次能力升级都有固定评估集和前后对比。
    • 工具调用有白名单、参数校验、权限和调用上限。
    • 应用具备测试、日志、安全、部署和回滚说明。

    文档完成标准¶

    后续每个课程单元都必须包含：

    • 前置知识。
    • 明确顺序。
    • 命令、示例或可操作步骤。
    • 练习或检查方法。
    • 常见错误与排查。
    • 客观验收标准。
    • 下一章节和可选并行分支。
