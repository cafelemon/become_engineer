# CS 核心

## 定位

CS 核心线用于解释工程背后的计算机科学基础。它不是Web之后才补的知识，而是先完成最小核心，再与Web实践并行深化。

本主线里的“算法”指通用CS算法，例如数据结构、复杂度、排序、查找、递归、动态规划和图。机器学习、深度学习、NLP和强化学习属于[AI 基础](../ai-foundation/README.md)里的模型算法。

## 适合人群

- 已经能写项目，但想理解底层原理的人。
- 准备提升后端、系统设计或性能优化能力的人。
- 想为 AI 工程和 LLM 系统打牢工程基础的人。

## 阶段划分

1. 最小核心：基础数据结构、通用算法、复杂度、进程线程、HTTP、网络和关系数据库概念。
2. Web并行深化：操作系统、TCP、HTTP、PostgreSQL/psql、MySQL对照、数据库事务、索引和并发。
3. 系统能力：缓存、队列、可用性、一致性、扩展性和安全。
4. 进阶选修：体系结构、分布式、编译原理和程序语言。

## 推荐学习顺序

1. Python/C++双主修达到阶段验收后，先学数据结构、通用算法、复杂度、HTTP和数据库最小集。
2. 最小核心完成后启动Web，不等待完整CS课程结束。
3. Web开发期间同步学习操作系统、网络、PostgreSQL优先的数据库原理、MySQL差异和并发。
4. 用真实项目的问题进入系统设计和更深原理。

## 进入与离开条件

- 进入条件：Python/C++双主修基础验收完成。
- Web解锁：能够解释常见容器复杂度、HTTP请求基本过程、PostgreSQL/psql基本查询和关系表基本操作。
- 本主线在Web、AI和LLM项目中持续深化。

详细顺序见[小白统一学习路线](../beginner-roadmap.md)。
