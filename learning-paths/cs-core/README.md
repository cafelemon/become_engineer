# CS 核心

## 定位

CS 核心用于解释程序、计算机和系统为什么这样运行。它不只服务Web或面试，也为系统开发、AI工程、LLM应用和[设备系统与边缘智能](../device-edge-systems/README.md)提供共同基础。

本主线里的“算法”指数据结构、复杂度和通用问题求解方法。机器学习、深度学习、NLP和强化学习中的模型算法属于[AI 基础](../ai-foundation/README.md)。

## 在默认路线中的位置

- 进入条件：Python起步完成，Python/C++达到核心语言和基础工程化验收。
- 必修主干完成后：解锁Web必修主干，并与CS推荐深化并行。
- 推荐深化和专业选修：不阻塞Web、AI或LLM/Agent默认路线，但可以成为更深选修的前置。
- 完整关系见[完整课程地图](../curriculum-map.md)。

## 适合人群

- 想理解数据结构、算法和程序性能的人。
- 准备工程师机考，同时不希望只背题的人。
- 想深入操作系统、网络、数据库、分布式或系统安全的人。
- 需要理解 MCU、嵌入式 Linux、PLC、设备通信或实时数据系统的人。

## 必修主干

必修主干负责建立共同概念和最小实践能力。学习者需要实现和验证基础结构，而不是只会说出名称。

| 模块 | 知识范围 | 实践与验收 |
| --- | --- | --- |
| 数据表示与复杂度 | 基本类型、内存与地址认知、数组、二维数组、字符串、时间与空间复杂度 | 能分析常见访问与遍历操作，完成边界输入测试 |
| 线性数据结构 | 链表、栈、队列及其数组或链式实现 | 能实现核心操作并解释适用场景和复杂度 |
| 哈希与集合 | 哈希思想、冲突认知、哈希表和集合 | 能完成字符统计、去重、频次和查找类问题 |
| 树与基础图 | 二叉树类型、存储方式、BFS、DFS、递归与迭代遍历，图的表示和基础遍历 | 能构造、遍历和验证树或图，并比较递归与迭代实现 |
| 查找与排序 | 顺序查找、二分查找、常见基础排序、稳定性和复杂度 | 能选择算法、实现代表性方法并用测试证明边界正确 |
| 递归与迭代 | 调用栈、终止条件、状态更新和递归转迭代 | 能定位无限递归、错误终止和循环边界问题 |
| 计算机运行基础 | 进程、线程、内存、文件、系统调用和程序执行过程 | 能解释程序从启动到读写文件的基本路径 |
| 网络最小核心 | 客户端/服务端、IP、TCP、DNS、HTTP、编码和序列化 | 能描述一次请求的路径并观察关键网络信息 |
| 数据库最小核心 | 关系模型、表、主键、SQL、索引和事务；PostgreSQL/psql优先，MySQL对照 | 能设计简单关系表并完成查询、修改和事务实验 |

### 指针与双指针的边界

- C++指针、引用、地址和生命周期属于语言与数据结构实现基础。
- 双指针是使用两个位置变量解决遍历或区间问题的算法技巧，不等于C++原始指针。
- 课程和题目必须明确当前讨论的是内存指针还是双指针算法，不能混用术语。

### 必修完成标准

- 能为数组、链表、栈、队列、哈希表和树的常见操作说明复杂度。
- 能用Python和C++分别完成若干代表性数据结构或算法实现，并记录语言差异。
- 能完成字符串、查找、排序、哈希、栈队列和二叉树的基础问题。
- 能解释递归、迭代、BFS和DFS的执行过程及边界。
- 能描述进程线程、HTTP请求和关系数据库的最小工作过程。

### 算法题型组织

题目按能力族组织，不为每个题目建立独立课程：

- 字符串：字符查找与统计、密码验证、排序、去重和模式处理。
- 栈与队列：括号、表达式、单调结构、火车进站和状态模拟。
- 贪心与区间：分发饼干、身高体重排序、区间选择和加油站。
- 二叉树：树的类型、存储、构造、层序、深度优先、递归和迭代遍历。
- 动态规划与回溯：状态定义、转移、边界、搜索空间和剪枝。
- 综合机考：读取输入、处理多组数据、复杂度控制、限时提交和失败复盘。

限时模拟可使用主力语言；代表性题目需要使用Python和C++对照实现，比较容器、类型、内存和标准库差异。

## 推荐学习顺序

1. 从已开放的 14 课开始，依次完成序列表示、线性结构、哈希集合以及查找排序四条双语言实验成果线。
2. 后续依次完成树与图、递归、系统、网络和数据库最小核心。
3. 解锁Web，同时选择算法与机考或一个CS推荐深化模块并行推进。
4. 在真实项目遇到性能、网络、数据库和架构问题时回到相应深化模块。
5. 满足全部前置后再进入设备系统与边缘智能、分布式、编译或安全专业选修。
