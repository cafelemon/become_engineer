# CS 核心

## 定位

CS 核心线用于补齐工程背后的计算机科学基础，包括数据结构、算法、操作系统、计算机网络、数据库原理和系统设计。

## 适合人群

- 已经能写项目，但想理解底层原理的人。
- 准备提升后端、系统设计或性能优化能力的人。
- 想为 AI 工程和 LLM 系统打牢工程基础的人。

## 阶段划分

1. 数据结构与算法：数组、链表、树、图、排序、搜索、动态规划。
2. 操作系统：进程、线程、内存、文件系统、并发。
3. 计算机网络：TCP/IP、HTTP、DNS、负载均衡、缓存。
4. 数据库原理：事务、索引、查询优化、存储。
5. 系统设计：可用性、一致性、扩展性、消息队列、缓存架构。

## 推荐学习顺序

1. 用算法和数据结构建立问题抽象能力。
2. 用操作系统和网络解释程序运行环境。
3. 用数据库和系统设计解释真实后端架构。
4. 用项目和案例把理论重新落回工程实践。
