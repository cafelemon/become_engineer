    Python 起步¶

    Python 起步的目标不是一次学完所有语法，而是建立完整的编程起点：会写小程序、会运行和拆分代码、会处理外部数据，也会根据错误与测试检查自己的判断。

    完成工程基础入门后再进入这里。你需要继续使用前面建立的习惯：每次学习都记录目标、代码、运行结果、错误和判断依据。

    前置要求¶

    开始前应完成：

    • 工程基础入门
    • 能打开项目目录或自己的学习目录。
    • 能使用终端运行简单命令。
    • 能用编辑器创建和保存文本文件。
    • 能写一份包含代码块和运行结果的 Markdown 学习记录。

    如果你还不能稳定找到文件、运行命令或记录错误，请先回到工程基础入门补齐。Python 起步会开始写代码，前置工具不稳会让后面的问题变得很难判断。

    学习目标¶

    完成 Python 起步后，你应该能做到：

    • 写出可以运行的 .py 文件。
    • 使用变量保存数据。
    • 理解数字、字符串、布尔值、列表、字典等常见类型。
    • 使用条件和循环控制程序流程。
    • 把重复逻辑整理成函数。
    • 读取文本或 JSON 文件，并输出简单统计结果。
    • 根据 traceback 定位到具体文件、行号和错误类型。
    • 使用标准库测试覆盖正常、边界和失败场景。
    • 为后续 Python 内容分析工具项目做好准备。

    学习顺序¶

    按下面顺序学习。七节课程已经全部落地，可以依次完成。

    课程 状态 你要掌握什么
    变量、基本类型、输入输出 已完成 写出最小脚本，理解变量、数字、字符串、布尔值、输入和输出
    条件、循环、布尔逻辑 已完成 使用判断和循环，让程序根据条件重复或分支执行
    函数、参数、返回值和作用域 已完成 把重复代码拆成职责清楚、可调用、可验证的函数
    字符串、列表、字典、集合和元组 已完成 根据顺序、可变性、访问方式和唯一性选择结构并处理多条记录
    文件、路径、JSON 和简单目录操作 已完成 定位和扫描文件，从 JSON 读取外部数据并生成文本报告
    模块、导入和虚拟环境 已完成 按职责拆分本地模块，控制导入副作用并使用项目虚拟环境
    异常、基本调试和最小自动化测试 已完成 定位和解释失败，为输入建立边界，并用自动化测试保护行为

    学习方式¶

    每节课都按可执行、可验证的方式推进：

    1. 先看本节要解决什么问题。
    2. 跟着示例写一个可以运行的脚本。
    3. 修改示例，观察输出变化。
    4. 完成练习，并记录代码、输入、输出和错误。
    5. 对照完成标准判断是否可以进入下一节。

    不要只复制代码。每段示例至少修改一次可观察行为，运行后记录变化；允许使用 AI 生成样板或重构候选，但必须人工审阅、解释和验证。

    运行方式¶

    本课程只使用 Python 标准能力，不需要第三方库。

    你可以把练习代码放在自己的学习目录中，例如：

    python-learning/
├── variables_io.py
├── conditions_loops.py
├── study_report.py
├── study_records.py
├── study_records_from_json.py
├── main.py
├── analysis.py
├── data_io.py
├── reporting.py
├── tests/
│   └── test_*.py
└── data/
    └── study_records.json

    常见运行方式：

    python3 variables_io.py

    Windows 上如果 python3 不可用，可以尝试：

    python variables_io.py

    如果命令不可用，不要跳过。请把完整错误写进学习记录，再回到工程基础入门的开发环境和验证习惯内容排查。

    完成标准¶

    完成 Python 起步需要满足：

    • 能独立创建并运行一个 .py 文件。
    • 能解释变量名和值的关系。
    • 能区分字符串、整数、浮点数和布尔值。
    • 能使用 if、for、while 写出基本分支和循环。
    • 能把重复代码提取成函数。
    • 能选择列表或字典保存多条数据。
    • 能读取一个文本或 JSON 文件，并输出统计结果。
    • 能根据 traceback 找到错误发生的文件和行。
    • 能为可预期输入错误给出清楚提示和非零退出码。
    • 能使用 unittest 和临时目录验证正常、边界和失败行为。
    • 能证明程序不修改输入文件，导入模块不启动主流程。
    • 能写出一份包含代码、输入、输出和判断的学习记录。

    满足以上标准后，完成并验收学习进度报告器。这个阶段作品整合连续五节课程，但不作为新的长期项目。

    项目解锁¶

    完成 Python 起步后，可以并行进入 Python 内容分析工具 的第一个里程碑。

    这个项目会用 Python 处理 Markdown、JSON 和目录数据，但不会一次要求你完成全部工程化内容。CLI、日志、配置和 CI 会随着双主修与工程化课程逐步加码。

    下一步¶

    尚未开始时，从变量、基本类型、输入输出开始。完成全部课程后，默认进入 Python / C++ 双主修，也可以并行启动 Python 内容分析工具 P1.1。
