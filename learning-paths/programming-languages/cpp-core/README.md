# C++ 核心语言

C++ 核心语言不是把 Python 语法换一种写法。它要求学习者开始理解编译、链接、静态类型、对象生命周期、资源管理和性能边界，并用编译器、测试和调试工具验证自己的判断。

本课程与 Python 深化按能力主题配对推进。不会机械地每门语言轮流一节；一个主题需要连续展开时，可以先完成同一门语言的必要上下文，再进入另一门语言的对应能力。

## 前置要求

开始前应完成：

- [Python 起步](../python-basics/README.md)及其阶段作品。
- 能在终端定位目录、运行命令并记录完整错误。
- 能解释变量、条件、循环、函数、输入输出和退出码的基本作用。
- 能使用 Git 和 Markdown 保存可复查的学习证据。

Python 起步提供了共同的编程概念，但 Python 经验不能代替 C++ 对类型、构建产物和资源生命周期的学习。

## 教学基线

- 语言标准：C++20。
- macOS/Linux：Clang 或 GCC。
- Windows：Visual Studio 2022 的 MSVC Build Tools。
- 起步阶段直接使用编译器命令，理解构建过程后再进入 CMake。
- 示例默认开启较严格警告，不把“成功生成可执行文件”当作唯一质量标准。

C++23及后续标准会在对应专题中说明，不使用预览特性作为核心课程前置。

## 学习方式

每节 C++ 课程都要求：

1. 保存源文件、编译命令、编译器版本和实际输出。
2. 开启课程指定的警告选项并逐条处理诊断。
3. 区分源文件、目标文件和可执行文件，不提交生成产物。
4. 至少制造并修复一次可稳定复现的错误。
5. 可以让 AI 生成骨架和诊断候选，但必须人工解释关键类型、生命周期和边界。
6. 修改至少一个可观察行为，并重新编译、运行和回归验证。

## 完成核心语言的标准

进入 C++ 工程化前，至少需要：

- 能解释源文件如何经过编译和链接形成可执行程序。
- 能使用显式标准和高警告级别编译代码。
- 能区分声明、定义、初始化、赋值和转换。
- 能使用函数、`std::string`、基础 STL 容器、迭代器和算法。
- 能解释引用、指针、对象生命周期和 RAII 的基本边界，并用受控文件导出验证资源释放。
- 能阅读常见编译与链接诊断，并形成最小复现。
- 能独立完成一个具有输入、输出、错误路径和验证证据的小型程序。
