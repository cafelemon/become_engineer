# C++：从起步到核心

C++ 路线服务系统工程、性能、设备软件和本地推理等方向。它不是把 Python 语法换一种写法，而是从“源码为什么要编译”开始，逐步进入静态类型、多文件构建、标准库、对象生命周期和资源管理。

现有五节课程已经按新目录分成两层，并全部迁移为 V2。两节起步课负责第一次编译、类型、输入输出和函数组织；三节核心课继续进入多文件 CMake、STL 多记录处理、对象生命周期与 RAII。

## 开始前

建议先完成：

- [Python 编程起步](../python-basics/README.md)及学习进度报告器。
- [CS 起步](../../cs-core/README.md)前四课，能够解释数据表示、位置和基本操作成本。
- 能在终端定位目录、运行命令，并保存完整错误信息。
- 能使用 Git 和 Markdown 留下代码、命令、输出和判断。

C++ 不是所有方向的必修。应用、Web、AI 或 LLM 学习者可以先走 Python 方向；准备系统、性能或设备方向时再进入这里。

## 工具基线

- 语言标准：C++20。
- macOS／Linux：Clang 或 GCC。
- Windows：Visual Studio 2022 的 MSVC Build Tools。
- 起步先使用编译器命令看懂构建过程，核心层再使用 CMake 和 CTest。
- 示例开启严格警告；生成可执行文件只是第一步，测试和诊断也要通过。

课程会分别给出安装、版本确认、编译命令和常见恢复方式。不要在没有记录编译器版本和完整诊断的情况下只说“运行失败”。

## 连续作品

五节课程继续演进[双语言学习进度报告器](../../../exercises/programming-languages/study-progress-reporters/README.md)：Python 与 C++ 使用相同的数据和主报告契约，C++ 侧逐步加入构建、容器、对象借用和 RAII 审计导出。

这个作品用来比较语言与工程选择，不要求两种实现逐行相同。学习者需要保存编译命令、测试结果、一次失败修复和一次主动修改。

## 学完后去哪里

- 系统工程：继续进入操作系统、并发、网络和性能。
- 设备系统：补 C 语言起步、系统基础和硬件接口，再进入 MCU/RTOS 或 Linux/BSP。
- 算法：进入共同算法基础和算法核心，用 C++ 完成代表性对照实现。
- 应用方向：可以回到 [Python 核心与工程化](../python-core/README.md)，不要求两条语言线机械轮换。

尚未开始时，从[源码到可执行程序](01-build-types-io.md)进入。
