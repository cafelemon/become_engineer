# CS 系统基础

这一组课回答一个很实际的问题：代码离开编辑器以后，操作系统、线程、网络和数据库到底替它做了什么。

六课已经全部开放，并共同形成“系统运行观察器”v0.1–v0.6。每课都包含正文、可运行示例、自动测试、小码知识库和故障恢复路径。

## 四类学习者怎么走

- **小白兴趣**：先完成前两课，再根据兴趣决定是否继续网络和数据库。
- **已有基础兴趣**：先运行每课的失败实验；能解释原因、修复并写测试时可以跳过正文。
- **小白求职**：六课按顺序完成，保存退出码、竞态、超时、事务与权限的故障复盘。
- **已有基础求职**：补未通过的检查，重点完成并发、网络、事务和安全追问。

Web 工程化需要其中的进程、网络、事务与权限前置；C++ 系统工程和设备系统还会继续深化内存、并发和性能。本课程不把 Python 线程实验包装成完整操作系统课程，也不把本机结果当作生产负载结论。
