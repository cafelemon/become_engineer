<section id="step-1" class="be-task-step" data-step-id="step-1" markdown="1">

### 第一步：用编辑器打开整个工作区

**任务：** 选择“打开文件夹/打开目录”，打开学习工作区而不是单个文件。**成功证据：** 文件树能显示 `notes`、`practice` 和 `assets`。

??? tip "提示一"
    单独打开一个文件会失去项目结构和相对路径上下文。
??? tip "提示二"
    如果文件树为空，检查是否误打开了 `notes` 而不是它的父目录。

</section>

<section id="step-2" class="be-task-step" data-step-id="step-2" markdown="1">

### 第二步：从文件树定位学习记录

**任务：** 展开 `notes` 并打开 `learning-log.md`。**成功证据：** 标签页显示正确文件，文件树中也能定位它。

??? tip "提示一"
    文件树负责定位，标签页负责在已打开文件之间切换。
??? tip "提示二"
    同名文件出现时，查看它所属的目录。

</section>

<section id="step-3" class="be-task-step" data-step-id="step-3" markdown="1">

### 第三步：完成一次修改并保存

**任务：** 在学习记录追加一行结果，保存后关闭再打开确认内容仍在。**成功证据：** 未保存标记消失，重新打开仍能看到新内容。

??? tip "提示一"
    修改出现在屏幕上不等于已经写入磁盘。
??? tip "提示二"
    使用保存快捷键后观察标签页上的圆点或状态提示。

</section>

<section id="step-4" class="be-task-step" data-step-id="step-4" markdown="1">

### 第四步：用搜索找到一个关键词

**任务：** 在工作区搜索“目标”或你写下的关键词，确认搜索范围。**成功证据：** 能说出搜索结果来自哪个文件。

??? tip "提示一"
    优先使用工作区搜索，而非只在当前文件搜索。
??? tip "提示二"
    没有结果时检查拼写、大小写和是否保存。

</section>

<section id="step-5" class="be-task-step" data-step-id="step-5" markdown="1">

### 第五步：在内置终端复查文件

**任务：** 打开编辑器内置终端，执行目录查看命令确认刚才保存的文件。**成功证据：** 编辑器和终端看到的是同一工作区。

??? tip "提示一"
    内置终端通常会从项目根目录启动，仍要先确认当前位置。
??? tip "提示二"
    如果目录不同，使用 `cd` 调整后再验证。

</section>

## 学习目标

学完本节后，你应该能用编辑器打开一个项目，理解文件树、标签页、搜索、修改、保存和内置终端的基本工作流。

本节采用平台中立写法。VS Code、Cursor、Trae 等现代编辑器界面不同，但基本概念相似。

本节不讲插件生态、调试器、AI 编程、复杂快捷键或主题美化。

## 核心概念

### 编辑器

编辑器是用来阅读、修改和组织代码或文档的工具。

刚开始学习时，不要把编辑器当成普通记事本。它至少承担三件事：

- 打开整个项目目录。
- 在文件树中定位文件。
- 修改文件并保存。

### 项目窗口

打开项目时，应该打开项目根目录，而不是只打开某一个文件。

推荐方式：

```text
打开 become_engineer/ 这个目录
```

不推荐方式：

```text
只打开 README.md 这个文件
```

只打开单个文件时，你看不到完整目录结构，也很难理解文件之间的关系。

### 文件树

文件树通常在编辑器左侧，用来展示项目目录结构。

看到文件树时，先做三件事：

1. 找到项目根目录名称。
2. 找到当前要学习的文件。
3. 判断这个文件属于路线、笔记、练习、项目还是文档。

### 标签页

打开文件后，编辑器通常会在上方显示标签页。

标签页表示你正在查看或编辑哪些文件。入门时不要同时打开太多文件，否则很容易迷路。

### 保存

修改文件后需要保存。

很多编辑器会用圆点、颜色或提示表示“文件已修改但未保存”。

入门时要养成两个习惯：

- 修改后立即保存。
- 保存后再验证结果。

### 搜索

搜索可以帮助你在项目中找到文件或文本。

常见搜索有两种：

- 文件名搜索：根据文件名打开文件。
- 全局文本搜索：在整个项目中查找某个词。

例如搜索 `工程基础入门`，可以找到路线、入口和相关课程文件。

### 内置终端

很多编辑器可以在窗口下方打开终端。

内置终端和系统终端本质上做的是同一类事情，只是它默认更容易处在当前项目附近。

入门时只需要会用它做三件事：

- 查看当前目录。
- 查看目录内容。
- 运行课程给出的简单命令。

## 学习顺序

1. 打开项目根目录。
2. 观察文件树。
3. 打开一个课程文件。
4. 修改一小段文本。
5. 保存文件。
6. 用搜索找到一个关键词。
7. 打开内置终端并查看当前目录。

## 示例：正确打开项目

假设项目结构是：

```text
become_engineer/
├── README.md
├── docs/
└── learning-paths/
```

正确做法是打开：

```text
become_engineer/
```

打开后，你应该能在文件树里看到：

```text
README.md
docs/
learning-paths/
```

如果你只看到 `README.md` 一个文件，说明你可能只打开了单个文件，而不是整个项目。

## 示例：搜索一个关键词

搜索关键词：

```text
开始学习
```

观察结果：

- 哪些文件出现了这个词。
- 哪个文件是正式入口。
- 哪些文件只是引用入口。

搜索不是为了快速跳过学习，而是为了理解一个概念在项目中出现在哪里。

## 示例：使用内置终端

在编辑器中打开内置终端后，先运行查看当前目录的命令。

macOS / Linux：

```bash
pwd
ls
```

Windows PowerShell：

```powershell
pwd
dir
```

确认它是否处在项目根目录附近。如果不是，先回到终端与 Shell 复习 `cd` 和当前目录。

## 实践练习

### 练习 1：打开项目根目录

用编辑器打开一个项目目录，并写下你在文件树中看到的前三个顶层项目。

需要产出：

```text
我打开的项目根目录是：

我看到的顶层文件或目录：
-
-
-
```

### 练习 2：打开课程文件

在文件树中找到本节文件：

```text
learning-paths/engineering-foundation/stage-0/04-editor.md
```

需要产出：

```text
这个文件所在目录是：

它的扩展名是：

它属于学习路线里的哪一部分：
```

### 练习 3：搜索关键词

在整个项目中搜索：

```text
工程基础
```

需要产出：

```text
我找到了哪些文件：
-

我认为哪个文件最适合作为工程基础入口：
```

### 练习 4：修改并保存

新建或打开一份自己的学习记录，写入一行：

```text
我已经能用编辑器打开项目、搜索文件并保存修改。
```

保存后重新打开文件，确认这句话还在。

需要产出：

```text
我修改的文件是：

我如何确认它已经保存：
```

### 练习 5：打开内置终端

在编辑器里打开内置终端，查看当前目录。

需要产出：

```text
内置终端显示的当前目录是：

它是否在项目根目录附近：

我的判断依据是：
```

## 常见错误与排查

| 错误 | 表现 | 怎么排查 |
| --- | --- | --- |
| 只打开单个文件 | 看不到项目结构 | 关闭文件，重新打开项目根目录 |
| 修改后没有保存 | 回到文件发现内容没变 | 看标签页是否有未保存提示，保存后再验证 |
| 搜索范围不对 | 搜不到明明存在的内容 | 确认打开的是整个项目，不是单个文件 |
| 文件开太多 | 不知道自己正在改哪个文件 | 关闭无关标签页，只保留当前任务相关文件 |
| 内置终端目录不对 | 命令找不到文件 | 先运行 `pwd`，再用 `cd` 进入正确目录 |

## 完成标准

完成本节需要同时满足：

- 能用编辑器打开项目根目录。
- 能通过文件树找到指定文件。
- 能搜索一个关键词并说出哪个文件最相关。
- 能修改并保存一个文本文件。
- 能打开内置终端并确认当前目录。
