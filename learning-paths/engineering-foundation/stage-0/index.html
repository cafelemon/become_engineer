    工程基础入门¶

    本课程帮助零基础学习者先建立基本学习方法和开发工作环境认知，再进入 Python 起步。

    目标不是一次学完所有工程能力，而是先具备继续学习的最低操作能力：知道自己在学什么、文件放在哪里、命令大概在做什么、出错时如何记录。

    前置知识¶

    无。只需要能打开电脑、浏览文件和编辑文本。

    学习顺序¶

    学习内容 状态 完成后能做到
    学习方法 已完成 把学习任务拆成可验证的小目标，并写出学习记录
    文件系统 已完成 解释目录、路径、扩展名、项目根目录、相对路径和绝对路径
    终端与 Shell 已完成 进入目录、查看文件、运行命令、读取退出状态
    VS Code 编辑器 已完成 安装并打开 VS Code，打开项目、搜索、修改文件、使用内置终端
    Markdown 已完成 用标题、列表、链接、代码块写清学习记录
    本地 Git 与 .gitignore 已完成 安装 Git，查看状态、忽略文件、暂存、提交和阅读历史
    GitHub 远程协作 已完成 解释本地与远程关系，完成 remote、push 和 clone
    开发环境 已完成 理解解释器、编译器、环境变量和依赖的基本概念
    Docker最小认知 已完成 解释镜像、容器、端口、挂载和环境变量的基本含义
    验证习惯 已完成 运行、观察输出、复现错误、记录命令和结果

    学习方式¶

    1. 按顺序学习，不跳过当前内容的练习。
    2. 每一节都留下一个可检查的产出。
    3. 遇到不懂的词，先记录问题，不急着扩展到高级资料。
    4. 完成当前内容后再进入下一节。

    完成验收¶

    完成全部内容后，应能做到：

    • 能创建一个本地 Git 仓库、配置 .gitignore、完成至少 3 次有意义的提交，并把练习仓库推送到 GitHub。
    • 能从终端进入项目目录并运行一个给定命令。
    • 能写一份包含目标、步骤、结果和问题的 Markdown 学习记录。
    • 能解释 Docker 解决的环境一致性问题，并知道 docker --version、docker run、docker ps、docker logs 和 docker stop 的用途。
    • 能根据错误信息指出失败发生在哪条命令，而不是只说“不能运行”。

    收口检查¶

    工程基础入门的 10 个主题按下面关系串联：

    1. 学习方法提供“目标、操作、结果、问题、下一步”的记录框架。
    2. 文件系统让后续命令、链接和项目根目录有共同语言。
    3. 终端与 Shell 让学习者能进入目录、查看文件和运行命令。
    4. 编辑器把文件树、搜索、保存和内置终端合到日常工作流里。
    5. Markdown 让每次学习都能沉淀成可读记录。
    6. 本地 Git 让文档和代码修改可以被忽略、保存、检查和回看。
    7. GitHub 远程协作让本地提交能够安全地 push，并能从远程 clone 验证。
    8. 开发环境解释命令、版本、PATH、依赖和虚拟环境为什么会影响运行。
    9. Docker最小认知提前建立环境一致性和部署工具的基本词汇。
    10. 验证习惯把前面所有能力合成“运行、观察、复现、记录”的闭环。

    本课程暂不展开 Pull Request、冲突处理、测试框架、CI、复杂 Dockerfile、docker compose、部署或安全加固。这些内容需要代码和项目上下文，会在后续 Python、Web、AI 和 LLM/Agent 项目中继续深化。

    下一步¶

    先完成学习方法，再依次进入后续内容。完成验收后，回到开始学习，进入 Python 起步。
