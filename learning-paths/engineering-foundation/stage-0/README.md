# 阶段 0：工程基础前置

本阶段帮助零基础学习者先建立基本学习方法和开发工作环境认知，再进入 Python 起步。

阶段 0 的目标不是把工程能力一次学完，而是让你具备继续学习的最低操作能力：知道自己在学什么、文件放在哪里、命令大概在做什么、出错时如何记录。

## 前置知识

无。只需要能打开电脑、浏览文件和编辑文本。

## 学习方式

1. 按顺序学习，不跳过当前单元的练习。
2. 每个单元都留下一个可检查的产出。
3. 遇到不懂的词，先记录问题，不急着扩展到高级资料。
4. 完成单元后再进入下一单元。

## 阶段验收

完成全部 9 个单元后，应能做到：

- 能创建一个 Git 仓库并完成至少 3 次有意义的提交。
- 能从终端进入项目目录并运行一个给定命令。
- 能写一份包含目标、步骤、结果和问题的 Markdown 学习记录。
- 能解释 Docker 解决的环境一致性问题，并知道 `docker --version`、`docker run`、`docker ps`、`docker logs` 和 `docker stop` 的用途。
- 能根据错误信息指出失败发生在哪条命令，而不是只说“不能运行”。

阶段验收以[小白统一学习路线](../../beginner-roadmap.md)为准。

## 阶段收口检查

阶段 0 的 9 个单元按下面关系串联：

1. 学习方法提供“目标、操作、结果、问题、下一步”的记录框架。
2. 文件系统让后续命令、链接和项目根目录有共同语言。
3. 终端与 Shell 让学习者能进入目录、查看文件和运行命令。
4. 编辑器把文件树、搜索、保存和内置终端合到日常工作流里。
5. Markdown 让每次学习都能沉淀成可读记录。
6. Git 让文档和代码修改可以被保存、检查和回看。
7. 开发环境解释命令、版本、PATH、依赖和虚拟环境为什么会影响运行。
8. Docker最小认知提前建立环境一致性和部署工具的基本词汇。
9. 验证习惯把前面所有能力合成“运行、观察、复现、记录”的闭环。

本阶段暂不展开测试框架、CI、复杂 Dockerfile、docker compose、部署、安全加固或协作流程。这些内容需要代码和项目上下文，会在后续 Python、Web、AI 和 LLM/Agent 项目中继续深化。
