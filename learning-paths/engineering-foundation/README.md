# 工程基础

## 定位

工程基础线用于建立长期学习和真实开发所需的底层工作习惯，包括 Git、命令行、编辑器、调试、测试、文档、自动化和问题拆解。

## 适合人群

- 刚开始系统学习工程开发的人。
- 已经会写代码，但缺少工程化习惯的人。
- 准备把个人学习过程沉淀成公开项目的人。

## 阶段划分

1. 工具入门：Git、Shell、编辑器、Markdown。
2. 开发习惯：分支、提交、README、目录结构、日志记录。
3. 调试测试：断点、日志、单元测试、命令行验证。
4. 工程协作：Issue、PR、代码审查、文档交接。
5. 自动化：脚本、格式检查、链接检查、基础 CI。

## 推荐学习顺序

1. 学会用 Git 管理公开仓库和私有资料边界。
2. 用 Markdown 写清项目目标、路线和进度。
3. 为每次学习建立可运行、可验证的最小产出。
4. 用脚本或检查清单减少重复劳动。
