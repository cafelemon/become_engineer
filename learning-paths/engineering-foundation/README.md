# 工程基础

## 定位

工程基础线用于建立长期学习和真实开发所需的底层工作习惯，包括 Git、命令行、编辑器、环境、Docker最小认知、调试、测试、文档、自动化和问题拆解。

本主线贯穿全部学习过程，但分为“语言前必修”和“项目中深化”两层。

## 适合人群

- 刚开始系统学习工程开发的人。
- 已经会写代码，但缺少工程化习惯的人。
- 准备把个人学习过程沉淀成公开项目的人。

## 阶段划分

1. 语言前必修：学习方法、文件系统、Shell、编辑器、Markdown、Git、环境、Docker最小认知和验证习惯。
2. 语言阶段深化：目录、依赖、调试、日志、异常和单元测试。
3. 项目阶段深化：分支、Issue、PR、代码审查、配置、Dockerfile和文档交接。
4. 交付阶段深化：CI、安全、docker compose、部署、监控和回滚。

## 推荐学习顺序

1. 在学习编程语言前完成文件系统、终端、编辑器、Markdown、Git、基本环境和Docker最小认知。
2. Python起步后加入调试、日志、异常和测试。
3. 进入项目后加入协作、配置、Dockerfile、自动化、安全和交付。
4. 每个阶段都保留运行命令、结果和错误记录。

## 进入与离开条件

- 进入条件：无，这是统一路线的第一个阶段。
- 语言解锁：能够从终端运行命令、使用Git提交、编写Markdown记录、解释Docker的用途并定位失败命令。
- 本主线不会结束；后续项目持续回链并深化。

详细验收见[小白统一学习路线](../beginner-roadmap.md)。
