# 课程内容规范 V2

## 目的与适用范围

本规范是 Become Engineer 正式课程、练习和项目内容的生产基线。它服务四类学习者：小白兴趣、小白求职、已有基础兴趣、已有基础强化求职。四类用户共享同一棵能力树；起点、补修和求职训练可以不同，知识事实和课程 URL 不复制。

十三个跨方向样板已于 2026-07-17 通过评审。55 节既有正式课程已经完成 V2 迁移，后续新建课程从创建时执行 V2；正式运行时继续保留 V1 读取能力，只用于历史或外部兼容数据，不再生产新的 V1 课程。

## 内容目标

- **先理解，再动手。** 第一屏先给结果、问题、界面或现象，让学习者知道为什么值得学。
- **例子要小，结果要真。** 代码和操作短到可以完整理解，并能在说明的环境中复现。
- **学习者必须修改。** 复制运行只是开始，还要改数据、规则或行为，并解释变化。
- **错误也是正文。** 写清最常见的失败现象、检查位置和恢复办法。
- **多节课共同做成作品。** 微型练习进入阶段作品，再沿方向项目持续演进。
- **深入不等于堆概念。** 原理、复杂度、性能、系统边界和求职内容在需要时进入。
- **AI 只提供待验证输入。** 学习者负责约束、审阅、修改、测试和最终判断。

## 共同语义骨架

V2 课程使用语义学习区域，不规定章节数量，也不要求按下面顺序排列。同一种类型可以出现多次。

| 类型 | 学习者在这里解决什么 | 要求 |
| --- | --- | --- |
| `overview` | 先看结果、问题、界面或现象 | 必需 |
| `concept` | 把关键关系想清楚 | 必需，可重复 |
| `example` | 用足够短的例子观察一件事 | 必需，可重复 |
| `reproduce` | 按环境、命令和结果复现 | 必需，可重复 |
| `modify` | 改成自己的数据、规则或行为 | 必需 |
| `troubleshoot` | 根据现象查找原因并恢复 | 必需 |
| `project` | 说明这节课进入哪条作品线 | 必需；无法接入时给出理由和等价产出 |
| `deepen` | 进入实现、性能、复杂度或硬件限制 | 可选 |
| `career` | 把代码、测试和复盘组织成求职表达 | 可选 |

每节课还必须让学习者找到：适合人群、前置、环境、学习目标、完成检查、来源与版本、下一步。

V2 正文区域示例：

```html
<section id="concept-variables"
         data-learning-context="concept-variables"
         data-context-type="concept"
         markdown="1">
```

`id` 与 `data-learning-context` 必须一致。顶部助教挂载点继续使用稳定 `lesson_id`：

```html
<div class="be-tutor-mount"
     data-tutor-lesson="lesson-id"
     aria-hidden="true"></div>
```

## 八类课程怎样组织

| 课型 | 适合的开场 | 主要组织方式 | 必须看到的结果 |
| --- | --- | --- | --- |
| 工具操作 | 完成后的真实界面 | 找入口、操作、确认变化、走错恢复 | 编号截图、文件或命令结果 |
| 编程起步 | 几行代码的实际输出 | 预测、运行、解释、修改、读错误 | 代码、输出和作品版本变化 |
| CS 概念 | 程序为什么这样运行 | 具体对象、表示、状态、抽象和限制 | 结构图、状态表和实现对照 |
| 算法与数据结构 | 先人工解决一个小问题 | 建模、轨迹、规律、实现、复杂度 | 可回放轨迹、边界测试和计数 |
| 系统与设备 | 故障、资源、生命周期或时序 | 系统边界、最小实验、机制、调试、可靠性 | 生命周期、时序、日志和仿真／真机说明 |
| AI 实验 | 数据、目标和基线结果 | 划分、基线、训练、验证、误差和复现 | 配置、指标、错误样本和结果对照 |
| Web／LLM／Agent | 请求、失败响应或受限工作流 | 契约、调用链、权限、评估和恢复 | 请求响应、状态、校验和离线替代 |
| 项目整合 | 当前版本和新增需求 | 设计、增量实现、测试、演示和复盘 | 版本差异、测试报告和诚实的项目叙事 |

课型决定教学组织，不改变课程在完整课程地图中的层级和前置。正式生产时使用公共共同模板，再读取课程生产 Skill 中唯一对应的课型差异模板。

## 语言基线

正文采用“老师陪练＋简洁教材”的表达。

- 技术定义短而准确；操作说明交代在哪里、会看到什么、走错后怎样回来。
- 标题简短，第一屏直接进入具体结果或问题，少写“本课将如何教学”的说明。
- 可以使用“这里先别急着”“我更建议”，但不能虚构作者经历、企业使用、用户量或性能结果。
- 避免整齐口号、连续排比和评审报告式语言。
- 面向学习者的正文不用“可观察结果、成功证据、迁移验收、受控失败实验、心智模型、本课增量、当前任务”等管理化表达。
- 技术领域中的“边界”正常使用，例如数组边界、权限边界和系统边界。

完成初稿后逐段朗读。判断标准不是是否像某个模板，而是学习者能否顺畅理解下一句话。

## 可复现代码与操作

完整示例必须注明：

- 文件名和语言。
- 操作系统、运行时或工具版本。
- 外部依赖和安装方式；只用标准库时明确说明。
- 从项目根目录执行的命令。
- 输入、预期输出或界面变化。
- 至少一个常见失败及恢复办法。
- 测试、检查或人工核对方式。

示例代码优先保存为独立源文件，再由正文引用，避免可执行代码和讲解漂移。不要用省略号隐藏关键逻辑，不放 Shell 提示符、真实密钥、私人路径或不可公开数据。

浏览器运行器只承载安全的微型代码；正式练习和项目仍在本地环境运行。C、C++、服务和设备课程使用真实工具链。付费 API、模型、硬件和高算力内容先给离线、Mock、仿真、小数据或低成本路径。

## 功能型视觉与交互

每张图只回答一个具体问题，并配等价文字说明。

- 工具课使用原创界面截图，标注平台、版本、核查日期和操作位置。
- 概念课使用关系图、状态表、生命周期图和实现对照。
- 算法状态难以静态表达时使用前进、后退、重置的轨迹，同时给出完整静态表。
- 系统、AI、LLM 和 Agent 课程展示数据流、状态、权限、失败路径和评估，不用装饰图代替机制。
- 颜色不是唯一信息；所有交互支持键盘、390px、深浅色和减少动画。
- JavaScript、CDN 或外部服务失效时，完整正文、代码、命令和静态说明仍可使用。

不设置“每课必须几张图”的数量指标。只有在视觉确实降低理解成本时才加入。

## 项目贯穿

每节进入连续作品线的课程回答五个问题：

1. 项目上一版已经会做什么？
2. 这节课新增了什么可以运行或观察的能力？
3. 哪些文件、接口或数据发生了变化？
4. 学习者需要保存什么代码、输出、测试、排错或解释记录？
5. 下一节准备继续增加什么？

每 3–6 节相关课程形成一个可独立运行、解释和验收的阶段作品。单模块作品放入 `exercises/`；推进长期项目时只增加里程碑，不重复建目录。

长期作品线包括：

- 学习进度报告器。
- 工程学习工作台。
- 跨语言可追踪数据引擎。
- 可复现实验与模型评估系统。
- 可评估的智能学习助手。
- 设备状态监测与边缘智能告警系统。

算法课程使用可追踪实验并迁移到方向项目，不强行把全部方向拼成一个超级项目。纯理论内容无法自然进入项目时，可以留下推理记录、实验结果或概念判断，但要说明原因。

## AI 协作与人工检查

AI 可以生成脚手架、测试候选、解释、图表和重构建议。学习者必须：

1. 说明目标、输入、输出和不能突破的限制。
2. 阅读生成的文件或差异，不直接接受全部修改。
3. 主动改变至少一个行为并运行验证。
4. 能解释关键逻辑和一次失败的排查过程。
5. 知道事实应该由官方资料、实验还是自动化测试证明。

以下内容不能只凭“能够运行”通过：

- 算法：状态变化、复杂度和边界用例。
- 内存与资源：对象生命周期、引用或指针关系、释放责任。
- AI：数据、划分、指标和决定结果的实验设置。
- Web／Agent：身份、授权、输入校验、工具白名单和密钥处理。

学习记录保存约束、AI 产出摘要、人工修改、验证结果和剩余风险，不要求公开私人对话。

## 小码 V1/V2 兼容契约

现有 V1 课程继续使用 `lesson + steps + cards`、`step_id` 和 `.be-task-step[data-step-id]`。V2 使用 `lesson + contexts + cards`、`context_id` 和语义正文区域。

- 单个知识库必须且只能包含 `steps` 或 `contexts` 之一。
- V1 本地状态使用 `be:tutor:v1:<lessonId>`；V2 使用 `be:tutor:v2:<lessonId>`。
- 浏览器只保存面板开关和提示展开层级，不保存原始提问。
- 每课至少 8 张审核知识卡、16 条固定问法、两层提示、局部例子和正文来源。
- 确定性检索最多返回 3 项，Top 3 不低于 80%；未知问题明确说没有可靠答案。
- 桌面使用大面积聊天窗，手机使用全屏面板；关闭后焦点回到入口。
- 正式迁移时页面、知识库和测试集在同一批次切换，URL 与 `lesson_id` 保持稳定。

V1 课程的 5–7 步只是兼容要求，不再是新课程的质量指标。

## 新手、深化与求职内容

课程不复制四份正文，而是组合：

- 共同课程：可靠知识、例子、修改、排错和项目。
- 新手补给：安装、找到软件、打开界面和第一次运行。
- 深化内容：原理、性能、复杂度和工程权衡。
- 求职训练：原创机考、项目证据问答、限时练习和复盘。

求职内容只通过课程生产 Skill 的脱敏查询脚本读取 V2 能力信号。受限平台的完整题面、答案、题解代码和个人信息不得进入课程素材。单一平台的“高频”只是平台信号；至少三个独立来源族且包含可靠开放来源，才能称为跨来源高频参考。

原创题重新设计场景、输入输出、约束、失败路径、测试和评分规则，不使用企业名称，不把外部题目换词后发布。

## 来源、版本与安全

每课必须列出：

- 适用的软件、语言、标准、模型或平台版本。
- 事实与界面的核查日期。
- 官方文档、标准、论文或一手资料。
- 代码、命令、输出、指标或真机结果的验证方式。

第三方博客只能帮助发现问题，不能替代关键事实的一手来源。外部模型和服务通过环境变量读取密钥；自动测试和站点构建不得调用真实 API，也不得记录原始请求、回复或工具参数。

设备课程坚持“仿真入门、真机毕业”。仿真只能证明平台无关逻辑；引脚、电气、时序、驱动、性能和可靠性必须由后续真机结果证明。

## 发布检查

- [ ] 已选择课型并满足相应差异要求。
- [ ] 第一屏从具体结果、问题、界面或现象进入。
- [ ] 七类必需语义区域完整，标题和顺序适合本课内容。
- [ ] 示例已经实际运行，环境、命令、输出和恢复办法一致。
- [ ] 学习者需要预测、修改、解释和排错，不只是复制。
- [ ] 项目上一版、本课变化、保存结果和下一步清楚。
- [ ] 来源、适用版本、核查日期和验证方式完整。
- [ ] 小码页面锚点、知识卡、固定问法和未知问题降级通过。
- [ ] 桌面、390px、键盘、深浅色、减少动画和无 JavaScript 路径可用。
- [ ] 已检查密钥、隐私、版权、费用、硬件和不安全代码。
- [ ] 课程登记、内部链接、示例测试、助教校验和 `mkdocs build --strict` 通过。
