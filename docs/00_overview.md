# 项目总览

## 项目定位

Become Engineer 是一个面向初学者的公开学习库，也是一套长期维护的个人学习工程。

对学习者来说，它提供一条可以照着走的学习路线：先建立工程基础，再进入编程语言、计算机基础、Web、AI 和 LLM/Agent。

对维护者来说，它提供一套内容治理方式：资料先经过筛选、验证、纠错、重组和实践化，再进入公开页面。

## 公开网站结构

公开网站只保留四个主入口：

- 首页：说明项目是什么、做什么、怎么开始。
- 开始学习：展示完整学习顺序和当前已开放课程。
- 项目：展示跨课程逐步完成的完整项目，以及练习和项目的关系。
- 项目文档：记录本项目的 PRD、路线图、架构、验收、进度和决策。

素材整理、资源评测、公开笔记和内部模板仍然保留在仓库中，但不作为公开网站主导航展示。

## 学习产品原则

- 对外优先使用学习者能理解的自然名称。
- 不把内部建设编号作为公开标题。
- 首页不展示仓库目录和素材处理流程。
- 学习路线只给出默认顺序，不要求读者自己拼装路径。
- 项目只在满足前置条件后解锁，并持续链接回基础课程。

## 内容边界

公开内容只展示可以被他人阅读、引用和复用的材料。

不公开：

- 私人学习记录。
- 未整理草稿。
- 账号、密钥、课程登录信息。
- 下载的受版权保护资料。
- 不适合公开的个人复盘。

公开内容需要经过整理、去隐私、版权检查和结构化重写。

## 当前重点

- 完成公开站点的信息架构收口。
- 保持首页、学习路线、项目入口和项目文档四个主入口清晰。
- 继续建设面向初学者的课程正文。
- 后续所有项目内容都围绕学习路线逐步解锁。

## 内容生产原则

- 先验证，再沉淀。
- 先结构化，再扩写。
- 保留来源，不搬运侵权内容。
- 每节课程都应让学习者知道前置、步骤、练习、错误排查、完成标准和下一步。
- 每个项目都要说明前置要求、里程碑、产出和验收方式。
