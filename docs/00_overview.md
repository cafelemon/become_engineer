# 项目总览

## 项目定位

Become Engineer 是一套面向四类学习者的公开工程学习系统：小白兴趣、小白转行求职、有一定基础的兴趣学习者，以及希望强化求职能力的有基础学习者。

对学习者来说，它提供一棵共享能力树：先建立工程基础、Python 编程和 CS 起步共同基座，再按目标进入应用工程、系统工程、算法、AI 模型、LLM 应用、Agent 工程或设备系统；求职目标在同一课程上叠加机考、项目证据和面试训练。

对维护者来说，它提供一套内容治理方式：资料先经过筛选、验证、纠错、重组和实践化，再进入公开页面。

## 公开网站结构

公开网站只保留三个主入口：

- 首页：由根目录 `README.md` 提供项目简介、作者与 GitHub 仓库信息，并给出第一课、学习路线和完整课程地图入口。
- 开始学习：直接进入第一节课程，并通过可折叠课程树继续后续学习。
- 项目：展示跨课程逐步完成的完整项目，以及练习和项目的关系。

`docs/` 中的项目治理文档继续在 GitHub 仓库维护，但不进入公开学习站点的导航、页面构建和搜索索引。

素材整理、资源评测、公开笔记和内部模板仍然保留在仓库中，但不作为公开网站主导航展示。

## 学习产品原则

- 服务对象固定为“小白／有基础 × 兴趣／求职”四类；不再用一条从极浅突然跳到强化内容的路线覆盖所有人。
- 55 节既有正式课程已经完成 V2 迁移；后续新课从创建时执行同一内容规范，按课型组织“理解驱动、小步实践、项目贯穿、分层深化”。V1 只保留运行时兼容，不再用于生产新课。
- 对外优先使用学习者能理解的自然名称。
- 不把内部建设编号作为公开标题。
- 首页不展示仓库目录和素材处理流程。
- 左侧导航只承担课程层级，右侧目录承担当前页面标题，避免长页面标题挤占课程导航。
- 开始学习页只解释四类入口、共同基座和方向摘要；完整课程地图是唯一公开课程控制面。
- 所有模块统一采用“起步、基础、核心、工程化、深化、专业/求职”六级深度，现有内容层级不合适时明确标记“已开放待重分层”。
- C++ 不作为所有人进入 CS 的强制前置，C 语言只进入设备方向；基础算法属于共同 CS 起步，算法核心与深化按方向或求职目标选择。
- AI 模型、LLM 应用和 Agent 工程分别建线；应用 Agent 不强制完整机器学习和深度学习，高级模型型 Agent 再追加相应前置。
- 项目只在满足前置条件后解锁，并持续链接回基础课程。
- AI 是默认学习与编程协作工具，学习者负责约束、审阅、解释、修改和验证。
- 课程按类型使用 Mermaid、表格、代码、输出和实验，丰富呈现必须服务于理解与复现。
- 实践分为正文内微练习、连续课程阶段作品和跨主线长期项目，不为每节课单独建项目。

## 内容边界

公开内容只展示可以被他人阅读、引用和复用的材料。

素材收集、来源候选、原始文件和清洗加工过程属于本地工作区：`content-inbox/`、`source-materials/` 和 `个人学习/` 整体由 Git 忽略，不进入 GitHub 或公开站点。公开仓库只保留独立整理后的课程、笔记、练习、项目、必要引用和项目治理文档。

不公开：

- 私人学习记录。
- 未整理草稿。
- 账号、密钥、课程登录信息。
- 下载的受版权保护资料。
- PDF、电子书、压缩包、音视频、Office 原件、大型数据集和模型权重。
- 不适合公开的个人复盘。

公开内容需要经过整理、去隐私、版权检查和结构化重写。

## 内容生产原则

- 先验证，再沉淀。
- 先结构化，再扩写。
- 保留来源，不搬运侵权内容。
- 每节课程都应让学习者知道为什么学习、关键关系、如何复现、如何主动修改、项目变化、错误恢复、完成检查和下一步；具体组织按课型选择。
- 每个项目都要说明前置要求、里程碑、产出和验收方式。
- 新增或扩写内容遵循[课程内容规范](08_content_standard.md)，AI 生成结果必须经过人工审阅和实际验证。
- 首页、学习路线、课程地图和课程页的渐进增强遵循[学习体验前端增强计划](09_learning_experience_frontend_plan.md)；正式课程均使用 V2 语义知识库，小码正式运行时继续保留 V1 `steps` 兼容能力，生产状态以[课程内容架构修订总计划](10_course_content_revision_plan.md)和台账为准。
