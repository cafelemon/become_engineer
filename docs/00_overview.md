# 项目总览

## 项目定位

Become Engineer 是一个面向公众的学习平台与知识库，也是一套服务个人长期成长的工程学习系统。

项目不追求简单堆链接，而是把来自免费资源的内容进行筛选、验证、纠错、重组和实践化，最终形成可阅读、可练习、可复盘、可持续演进的公开学习资料。

公开学习内容首先面向从零开始的学习者。读者应能沿着统一顺序逐项操作、练习和验收，而不是先自行理解六条主线如何组合。

## 核心目标

- 建立覆盖工程与 AI 的长期学习地图。
- 收集高质量免费资源，并记录来源、适用阶段、优缺点和使用建议。
- 将零散知识沉淀为结构化笔记、路线图、练习题、项目任务和复盘记录。
- 通过公开仓库形成可被他人学习、审阅和贡献的知识产品。
- 保留个人学习空间，但不把私人资料暴露到公开仓库。

## 四层六线

### 四层结构

- 学习地图：说明全局方向、顺序和能力关系。
- 学习路线：拆解每条主线的阶段目标和推荐路径。
- 项目实践：用练习和项目验证真实掌握程度。
- 资源评测：筛选、纠错、解释资源适用阶段和使用建议。

### 六条主线

- 工程基础
- 编程语言
- Web 全栈
- CS 核心
- AI 基础
- LLM/Agent

六条主线是内容分类。唯一默认学习顺序为：

1. 工程基础前置。
2. Python 起步。
3. Python/C++ 双主修，Java作为补充。
4. CS 最小核心。
5. Web 实践与 CS 深化并行。
6. AI 基础。
7. LLM/Agent。

## 内容边界

### 公开内容

- 学习路线与阶段目标
- 主题笔记与知识卡片
- 资源清单与评估
- 可公开的代码示例
- 项目练习与实验记录
- 错误修订与版本说明

### 私有内容

- 个人原始学习笔记
- 未整理草稿
- 账号、密钥、课程登录信息
- 下载的受版权保护资料
- 不适合公开的个人复盘

私有内容统一放入本地 `个人学习/` 目录，并由 `.gitignore` 排除。

## 首批学习领域

- 工程基础：Git、命令行、环境、Docker最小认知、调试、测试、工程习惯。
- 编程语言：Python、C++ 双主修，Java补充；JavaScript/TypeScript在Web阶段学习。
- Web 全栈：前端、后端、PostgreSQL/psql、MySQL对照、API、部署。
- CS 核心：数据结构、通用算法、操作系统、网络、数据库原理、系统设计。
- AI 基础：机器学习、深度学习、NLP/Transformer、强化学习等模型算法。
- LLM/Agent：Prompt、RAG、工具调用、评估、Agent 项目。

## 内容生产原则

- 先验证，再沉淀。
- 先结构化，再扩写。
- 保留来源，不搬运侵权内容。
- 公开内容面向他人阅读，个人草稿保留在私有区。
- 每个主题尽量包含概念、路线、资源、练习和项目。
- 每个课程单元都应让小白知道前置、步骤、练习、错误排查、验收和下一步。
- 项目只在满足前置条件后解锁，并持续链接回基础课程。
