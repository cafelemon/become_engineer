# 产品需求文档

## 背景

工程与 AI 学习资源很多，但常见问题是路径混乱、质量参差、内容重复、实践不足、缺少复盘。这个项目希望把免费资源整理为一套可长期使用的公开学习平台，让学习者能够从零开始，并逐步走向工程实践、专业应用和深入理解。

## 用户

### 主要用户

- 项目作者本人：用于系统学习、复盘、查漏补缺和项目实践。
- 小白 · 兴趣学习：需要从软件在哪里、怎样打开开始，完成共同基座后自由选择方向。
- 小白 · 转行求职：需要完整共同基座，并在阶段验收后逐步加入机考、项目证据和面试训练。
- 有一定基础 · 兴趣学习：需要跳过已证明掌握的内容，补具体缺口后进入核心、深化和兴趣项目。
- 有一定基础 · 强化求职：需要能力缺口补修、方向深化、限时题、故障复盘、项目追问和岗位专项。

### 次要用户

- 想快速了解某一技术栈的人。
- 想贡献优质免费资源或修订错误的人。

## 用户需求

- 我想知道从零到可实践某个方向应该怎么学。
- 我希望按文档从第一步开始操作，不需要先判断课程顺序。
- 我想找到经过筛选的免费学习资源。
- 我想知道每个资源适合什么阶段、有什么坑、如何使用。
- 我想通过练习和项目验证自己是否真的掌握。
- 我希望知道如何用 AI 提高编程和学习效率，同时保留自己审阅、修改、排错和验证的能力。
- 我希望课程不只是纯文本，而是用图表、可复制代码、运行输出和实验帮助我理解并复现。
- 我希望课程不止帮助我入门，还能告诉我如何继续进入工程化、运行原理、性能和专业领域。
- 当我需要 MCU、嵌入式 Linux、工业控制或端侧 AI 时，我希望看到跨 C++、CS 和 AI 的完整前置、硬件门槛与能力出口；没有设备方向需求时可以完全跳过。
- 我想看到学习内容的来源、版本和修订记录。

## 核心功能

- 首页导航：集中提供项目介绍、默认学习路线和完整课程地图入口。
- GitHub 引导：首页展示作者、项目仓库和克制的 Star 引导，项目治理文档只在仓库中查看。
- 直接开课：点击“开始学习”后进入第一节课程，通过可折叠课程树继续学习。
- 双栏目录：左侧显示课程层级，右侧显示当前页面标题目录。
- 学习地图：按共同基座、方向分流、跨线汇合和前置关系组织内容。
- 开始学习页：只提供四类用户入口、共同基座和方向摘要，不复制维护完整课程结构。
- 完整课程地图：作为唯一公开课程控制面，展示可点击状态流程、阶段总表、55 节明细与规划模块边界。
- 设备系统与边缘智能：作为后置跨主线专业领域，提供设备共同基础、MCU/RTOS、Linux/BSP、工业控制、边缘智能与智能控制的解锁关系。
- 单一总课程表：四类画像共用同一课程 URL 和正文，通过基础课程、新手操作包、深化挑战包、求职训练包组合出不同路线，不维护四套重复课程。
- 资源库：收集免费资源并做质量评估。
- 笔记库：沉淀公开可读的知识笔记。
- 练习库：提供题目、实验和项目任务。
- 课程内容规范 V2：以七类语义学习环节和八类课型差异统一 AI 协作、自然语言、功能型视觉、可复现代码、项目贯穿和人工检查要求。
- 三级实践：单节微练习、连续课程阶段作品和跨主线长期项目逐级汇聚。
- 正式课程兼容：55 节已开放课程已经按连续课程组迁移为 V2，URL、稳定课程标识和学习进度隔离保持不变；运行时继续支持历史 V1 `steps` 数据，不做机械批量重写。
- 正式迁移台账：逐课登记目标层级、角色、课型、前置、项目线、稳定小码 ID、批次和迁移状态；页面、知识库与测试必须在同一批次切换。
- 课型化内容体验：后续按工具、编程、CS、算法、系统设备、AI、应用工程和项目整合八类组织“理解驱动、小步实践、项目贯穿、分层深化”，具体路线见[课程内容架构修订总计划](10_course_content_revision_plan.md)。
- 智能方向样板必须区分 Web 请求、机器学习实验、LLM 结构化输出和 Agent 工具调用；模型输出与工具参数均视为不可信输入，离线示例是默认学习和自动验收路径，真实模型调用只作本机可选验证。
- 知识库助教：在全部已开放正式课程提供不依赖大模型的上下文问答、分层提示和可靠来源。
- 课程生产 Skill：支持新建 V2、V1 迁移到 V2 和维护 V1 三种模式，按课型同步产出正文、语义知识库、检索测试和站点接入。
- 求职参考素材生产链：将公开来源按自动采集、人工浏览和禁止采集分级，在本地归纳能力信号并独立设计机考与面试题；V2 使用 600 条信号区分方向参考、跨来源高频参考和课程重点，参考样本不直接成为公开题库。
- 纠错机制：记录错误、争议、修订来源和结论。
- 进度记录：记录项目建设过程与下一步任务。

## 非目标

- 不直接搬运付费课程或受版权保护内容。
- 不镜像平台题库、用户面经、题解代码或企业内部材料，不以社区来源给原创题标注企业归属。
- 不做只堆链接的导航站。
- 不在公开仓库保存个人隐私学习资料。
- 初期不追求复杂账号系统、社交系统或在线判题系统。
- 设备专业路线首版规划不采集专项素材、不指定开发板、不把仿真结果包装为真实硬件验收，也不复制一套新的默认课程主线。
- 知识库助教 V1 不接模型 API，不保存完整对话，不提供跨设备同步。

## 小码入学测评与个性化路线 V1

公开结果使用“从零起步／已有基础 × 兴趣学习／求职准备”四类主画像。系统内部同时保留工程工具、编程、算法与 CS、项目工程四个 0–2 级能力维度；主画像负责让用户快速理解方向，能力维度负责补修与跳课。

用户旅程固定为：首次可跳过邀请 → 选择学习目的 → 完成八道能力情境题 → 查看画像、依据、建议起点和全部未显示原因 → 主动确认个性化导航。确认后仍可逐项加入内容、重新测评或切换完整路线；搜索和直接 URL 永远允许访问全部公开内容。

四类路线分别为完整基础兴趣路线、缺口补修兴趣路线、完整基础求职路线和缺口补修求职路线。未显示原因固定为已掌握、目标不匹配、前置未满足、内容未建设四类，不允许静默隐藏。

完整课程地图同时承担总课程规划：公共基座固定为“工程基础起步 → Python 编程起步 → CS 起步 → 首个可验证项目与方向选择”，之后分为应用工程、系统工程、算法、AI 模型、LLM 应用、Agent 工程和设备系统。后续新增资产优先挂到既有模块和课程证据，不因画像不同复制正文。

模块统一记录六级深度、模块角色、前置、解锁、四类用户规则、内容状态和衔接状态。现有课程若可访问但层级或前置不适合新架构，必须标为“已开放待重分层”，不得伪装成顺畅主线。

V1 只在浏览器本地保存画像、能力等级、确认状态、导航模式和用户主动加入的模块；不保存原始答案、自由文本、账号、云端历史或跨设备标识。存储或 JavaScript 失效时回退到完整默认路线。

## 成功标准

- 仓库结构清晰，新读者能在 5 分钟内理解项目目标与入口。
- 新读者能根据统一路线确认当前阶段、必学内容和下一步。
- 四类学习者都能从课程总表判断自己的起点、可跳过内容、兴趣方向和求职叠加内容。
- 机器学习、深度学习、强化学习、LLM 应用和 Agent 工程的关系无歧义；应用 Agent 不被错误绑定到完整模型路线。
- 新读者点击“开始学习”后无需再次选择路线即可进入第一节课程。
- 公开网站只显示首页、开始学习和项目三个顶部入口，`docs/` 不生成站点页面且不进入搜索索引。
- 首页、学习路线、完整课程地图和课程样板使用同一套克制、可访问、兼容深浅色的视觉语言。
- 桌面端课程树可以按阶段展开和收起，长页面标题不会混入课程树。
- 课程树的大章节与具体课名通过字号、字重和颜色形成清楚但不过度装饰的视觉层级。
- 每个公开主题都有来源、学习目标、核心笔记和练习建议。
- 课程学习环节包含方向与结果预览、功能型视觉、微型例子、可复现操作、主动修改、常见错误、项目增量和明确验收。
- 编程、算法、Web、AI/LLM和项目课程达到各自的呈现与复现最低要求，不机械套用同一种格式。
- AI 生成代码经过人工审阅、实际运行、关键逻辑解释、主动行为修改和验证，不以对话结果代替学习证据。
- 可复制示例包含文件、环境、依赖、命令、预期输出、失败路径和检查方法。
- 阶段作品默认由连续3-6节课程形成，并进入练习或既有项目里程碑，不恢复“一课一个项目”。
- 现有正式课程继续保留 5–7 步、主动修改和失败路径；V2 不再把固定步骤数作为跨课型质量指标。
- 新样板按课型组合理解、示例、复现、修改、排错、项目和深化环节；第一屏不得用密集任务列表替代方向感和最小心智模型。
- 样板批次 A 至少邀请四类目标用户各 3 人验证理解、复现、主动修改和项目增量，不用页面停留或点击次数代替学习效果。
- 每节课程知识库至少 8 张审核卡、16 条固定问法，Top 3 命中率不低于 80%。
- 助教对未知问题必须明确降级，不得拼接、猜测或伪装生成答案。
- 主要能力路线同时说明入门、进阶、工程化、深入和领域应用，不把“学过语法”当成掌握一门语言。
- 专业分支说明用途、前置、学习顺序、跳过规则和实践终点，不强迫无相关需求的学习者完成。
- 设备系统与边缘智能明确采用“仿真入门、真机毕业”：一级基础实践允许仿真，二级独立模块开发和三级 BSP／边缘智能高级必须提交真实设备证据。
- AI 模型、LLM 应用与 Agent 工程保持独立能力线；边缘智能需要 AI 模型基础与设备平台能力，但强化学习不是其通用前置。
- 所有模块采用六级深度和四类角色；能力深化不阻塞其他方向，但可以成为专业模块的前置。
- 算法机考作为CS推荐深化的验收体系，覆盖基础实现、题型练习、限时模拟和错题复盘，不单独包装为求职课程。
- 求职参考素材的每条信号都能追溯到来源、访问日期、许可或条款状态和独立能力摘要；受限来源的完整题面、答案和个人信息保存数量为零。
- V2 精确包含 320 条机考和 280 条技术面试／项目追问信号，至少覆盖 18 个来源和 8 个独立来源族；单一来源不超过总量 20%，在单一能力类别内不超过 35%。
- “高频参考”至少需要三个独立来源族且包含 A 级来源；课程重点另行叠加前置中心性、岗位覆盖、项目价值和课程缺口，不用单一平台次数冒充行业频率。
- 原创机考与面试题必须重新设计场景、约束、测试和完成证据，并在接入公开课程前完成相似性复核与学习者试做。
- 私有目录不会被 git 跟踪。
- 资源引用符合版权和开源边界。
- 项目可以持续按版本迭代。
