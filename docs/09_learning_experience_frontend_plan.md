# 学习体验前端增强计划

## 目标

Become Engineer 的内容已经具备明确的前置、步骤、练习和验收，但首页、学习路线、课程地图和课程正文仍以连续文本为主。本计划通过渐进式前端增强，让学习者更容易判断从哪里开始、当前处于哪个阶段、下一步是什么，以及必修、推荐深化和专业选修之间的关系。

增强优先服务学习方向感、阅读节奏和反馈，不把文档站改造成营销首页，也不以动画、装饰图片或大量卡片代替扎实内容。

## 技术结论

- 保留 MkDocs Material、Markdown 内容源、现有 URL 和 GitHub Pages 发布方式。
- 第一阶段优先使用现有 Markdown 扩展、语义化 HTML 和 CSS，不新增前端框架。
- 只有课程地图筛选等确有状态的交互才引入少量原生 JavaScript，并保证 JavaScript 失效时完整内容仍可阅读。
- 不在 V1 引入账号、后端、云端学习记录、在线判题或多设备同步。
- 不批量改写全部既有课程；先完成样板，验收后再更新模板并逐步迁移。
- 55 节已开放正式课程允许加载公开 JSON 与原生 JavaScript，实现不依赖模型的上下文知识库助教。
- 新课程内容体验以[课程内容架构修订总计划](10_course_content_revision_plan.md)为唯一目标路线；本计划只负责其视觉、交互和渐进增强边界，不重复维护课程样板清单与迁移顺序。

## 设计基线

学习路线阶段 Stepper 已于 2026-07-14 通过人工评审，成为本项目入口、地图和课程页面的正式视觉基线。基线强调清晰方向、克制卡片、可见状态、充足留白和完整正文，不采用营销式大图、装饰动画或复杂前端状态。

### 信息层级

- 顶部首先回答“从哪里开始”，再提供路线和课程地图入口。
- 学习路线按阶段顺序展示状态、目标和入口。
- 课程地图使用“必修主干、推荐深化、专业选修”三种明确层级，颜色之外同时保留文字标签。
- 课程页突出所属阶段、前置、学习目标、练习、完成标准和下一课。

### 视觉与交互

- 继续使用项目现有靛蓝主色，青绿色表示推荐深化，琥珀色表示专业选修。
- 卡片只用于阶段、状态和可点击入口，不给每段正文套卡片。
- 圆角、阴影和动画保持克制；支持深浅色模式及 `prefers-reduced-motion`。
- 可点击区域支持键盘焦点，触摸目标不小于 44px；状态不能只依靠颜色表达。
- 桌面端和手机端使用同一语义结构，布局在窄屏下转为纵向，不依赖水平滚动。

## 版本范围

### V1.0：视觉基础

- 建立页面入口、阶段 Stepper、状态标签、课程层级、按钮和焦点样式。
- 明确浅色、深色、移动端和减少动态效果规则。

### V1.1：首页与学习路线

- 首页突出“开始第一课”，并提供学习路线和完整课程地图次级入口。
- 学习路线改为可点击阶段 Stepper，展示已开放、建设中和规划中状态。
- 每个阶段跳转到当前页面的详细说明；已开放阶段额外提供课程入口。

### V1.2：完整课程地图

- 首版使用静态视觉图例、领域入口和三层内容卡片，先解决方向感与可读性。
- CS、Web、AI、LLM/Agent 保留完整正文、前置、验收和跳过影响。
- 筛选与展开需要状态管理，留到三页样板通过后另行设计；本轮不引入 JavaScript。

### V1.3：课程阅读样板

- 选择《学习方法》作为首个课程样板。
- 增加课程状态、为什么学习、目标清单、核心流程、可展开提示、练习反馈、完成标准和下一课入口。
- 样板验收后更新课程模板；旧课程只在后续扩写时迁移。

### V2：本地学习状态

- 在 V1 验收后再评估完成标记、学习进度和“继续上次学习”。
- 第一版仅使用浏览器本地存储，课程 URL 作为稳定标识。
- 多设备同步需要账号和后端，必须另行立项。

### V1.4：任务驱动课程与知识库像素助教

- Python 第一课已改造为七步连续任务，并以个人学习档案贯穿变量、类型、输入输出、转换和错误定位。
- 右下角原创像素宠物打开聊天面板，支持推荐问题、自由输入、前三候选和逐层提示。
- 回答来自已审核结构化知识卡；未知问题明确降级，不生成内容。
- 本地只保存提示层级和面板状态；首版不登录、不保存完整对话、不跨设备同步。
- 样板通过验收后，任务版曾推广到 55 节课程；未迁移页面继续保留 5–7 步与至少 8 张知识卡、16 条固定问法，避免兼容中断。
- 正式运行时兼容 V1 步骤与 V2 语义上下文。工程基础 10 节和 Python 起步 7 节已经切换到 V2，其余课程按连续批次迁移，不能再把任务版描述成未来内容目标。

### V3：课型化课程内容样板

- 样板批次 A、B、C 和十三个样板跨方向统一评审均已通过并冻结，老师陪练语言、结果开场、功能型视觉、小例子和静态降级成为正式基线。
- 智能方向的浏览器演示只负责显示状态；FastAPI、训练、结构校验和工具执行均有独立可运行源文件，禁用 JavaScript 后保留完整静态讲解。
- 互动播放器只用于关键状态变化并保留静态降级；Python 浏览器微型运行器按需加载，不能替代本地工程证据。
- 内容结构和小码双协议已经固化；批次视觉与互动组件随首个正式迁移课程组按需提取，不一次性搬入生产样式。

## 首个样板

首个样板改造[学习路线](../learning-paths/beginner-roadmap.md)顶部总览：

- 用纵向阶段 Stepper 替换纯文本路线树。
- 阶段卡展示序号、名称、简短目标和状态。
- 点击阶段卡跳转到本页对应详细章节。
- 早期样板将工程基础和 Python 起步标记为“已开放”、Python/C++ 双主修标记为“建设中”；该单线表达已被课程架构 V2 取代。
- 可选分支继续保留在路线总览下方，但不抢占默认主干的视觉优先级。
- 页面下方现有课程说明、前置、内容和验收保持不变。

该样板已通过评审。本轮按同一视觉语言推进 README 首页、完整课程地图和《学习方法》课程正文，形成三类可复用样板；其他课程不批量迁移。

课程架构 V2 继续复用这套视觉语言，但把学习路线收缩为四类入口、共同基座和方向摘要；完整课程地图升级为唯一课程控制面，并用响应式状态流程和阶段卡片展示分流、汇合、建设状态与衔接缺口。

## 本轮三页样板

- README 首页：紧凑入口面板、项目简介、作者与 GitHub/Star 引导。
- 完整课程地图：三层图例、领域锚点和清晰的内容层级，不增加筛选状态。
- 《学习方法》：课程定位、目标、完成证据和下一步入口，正文内容保持完整。
- 三页共享颜色、边框、圆角、按钮、焦点、响应式和减少动画规则，细节以 `.ai/design/system.md` 为准。

## 验收标准

- 新学习者能在 5 秒内识别默认路线的起点和顺序。
- 七个阶段都能通过鼠标、键盘或触屏访问对应详细章节。
- 已开放、建设中和规划中同时使用文字、颜色和视觉形态区分。
- JavaScript 禁用时路线与全部正文仍可阅读。
- 1280×720、1440×900、1920×1080 和 390×844 下无页面横向溢出。
- 浅色、深色和减少动态效果模式下信息仍完整清晰。
- `mkdocs build --strict` 通过，现有课程 URL、搜索、导航和上一页/下一页不回归。

## 推进与回退

- 每个版本只处理一个页面族，完成桌面、移动端、键盘和构建检查后再扩展。
- 课程内容样板每次只处理总计划中的一个批次；批次未通过评审时修正当前批次，不提前建设下一批或批量迁移正式课程。
- 页面增强只改变呈现，不删除原始学习内容和链接。
- 若样板未通过评审，回退对应 Markdown 结构和 CSS 组件即可，不影响课程正文和发布架构。

## 首页测评与个性化导航样板

- 首页小码使用独立大面板完成目的选择、八道能力情境题、四维结果和路线确认；移动端全屏，桌面端保持大面积内容区。
- 个性化模式在每页显示画像、未显示数量与完整路线切换；当前路线外页面只给状态提示，不遮挡正文。
- 未显示清单使用文字和状态共同区分已掌握、目标不匹配、前置未满足与未建设，支持逐项加入。
- 所有交互可用键盘完成；减少动画时宠物固定首帧；脚本或存储失败时回退完整导航。
