# AI Coding Agent 工作协议

## 总原则

这个项目优先追求最终学习效果，而不是拘泥于当前工具是否已经具备。需要新工具、依赖或插件时，AI Coding Agent 应说明原因并申请安装或授权。

## 项目启动规则

新项目或新模块先写文档，再写实现。文档至少包含：

- 项目总览
- 产品需求
- 版本规划
- 技术架构
- 验收清单
- AI Coding 工作协议
- 进度记录
- 决策记录

## 内容边界

- 公开内容必须适合进入 GitHub。
- `个人学习/` 是本地私有目录，不得提交。
- 不把账号、密钥、课程登录信息、下载资料原件写入公开仓库。
- 不直接搬运受版权保护内容。

## 工作方式

- 先理解当前仓库结构，再修改文件。
- 优先保持文档、目录和命名的一致性。
- 所有课程、笔记、练习和项目先定位到[面向小白的统一学习路线](../learning-paths/beginner-roadmap.md)，不得绕过前置阶段。
- 六条主线用于组织内容，不得被解释为六条可任选的起步路线。
- 外部来源已登记不等于课程已完成；按当前阶段只取最小必要内容并独立重写。
- 对学习资源做筛选、归类、纠错和摘要，而不是简单堆叠。
- 遇到不确定知识点时标记为待验证。
- 对外发布前检查隐私、版权和准确性。

## 推荐输出格式

资源条目应尽量包含：

- 名称
- 链接
- 类型
- 语言
- 适合阶段
- 前置知识
- 推荐理由
- 主要风险或缺点
- 学习建议
- 验证状态

学习笔记应尽量包含：

- 学习目标
- 前置知识和统一路线位置
- 可顺序执行的步骤
- 核心概念
- 常见误区
- 常见错误和排查方式
- 实践练习
- 客观验收标准
- 下一章节或并行分支
- 延伸资源
- 自测问题
