## 2026-07-16: 断开图以最小生成森林为正式结果

### 决策

CS 核心第 24、25、26 课新建“可追踪生成森林实验”。并查集使用完整路径压缩与按大小合并；Kruskal 固定边排序；Lazy Prim 从最小未访问顶点重启分量，并在弹出时跳过过期边。

### 原因

断开图没有单棵生成树，但每个连通分量仍有最小生成树。把最小生成森林作为共同契约，可以让孤立点、负权边和相同权重都被确定性测试；同时避免把 DSU 父数组误解为原图路径，或要求 Kruskal 与 Prim 选择完全相同的边。

### 影响

- DSU 只维护集合关系，不保存图路径，也不直接支持边删除；多次操作按摊还 `O(alpha(n))` 表述。
- 相同大小合并由较小根保留代表身份；相同权重边按 `(weight,u,v)` 破除实现层平局，但不据此宣称最优森林唯一。
- 两种森林算法只比较总权重、分量数和 `V-C` 边数；总权重按有符号 64 位安全累加。
- 本轮不进入 Boruvka、动态图、可回滚并查集、MST 唯一性、负权最短路、有向图、网络流或正式机考。

## 2026-07-16: 优先队列使用稳定序号，Dijkstra 使用懒惰重复项

### 决策

CS 核心第 21、22、23 课新建“可追踪优先队列与最短路实验”。整数最小堆使用零基隐式树；任务优先队列以递增序号稳定处理相同优先级；Dijkstra 不实现 `decrease-key`，获得更短距离时压入新项，并在弹出时跳过过期距离。

### 原因

局部堆不变量、优先队列接口和最短路算法是三个不同层次。独立记录序号可以让同优先级结果跨语言确定；懒惰重复项符合 Python `heapq` 与 C++ 容器适配器缺少直接降权接口的现实，同时能用过期项计数呈现正确性边界。

### 影响

- 堆数组只承诺父节点不大于孩子；查看堆顶为常量时间，插入和删除为对数时间，自底向上建堆为线性时间。
- 带权图拒绝负权、自环、重复边和越界端点；等距候选不覆盖首次父节点，距离加法超过有符号 64 位上限时受控失败。
- 教学实现复杂度按懒惰重复项模型表述为 `O((V+E) log E)`，不套用可变键堆的复杂度结论。
- 本轮不进入最大堆、堆排序、Top-K、可索引堆、负权算法、最小生成树、并查集、有向图算法或正式机考题。

## 2026-07-16: 一般图基础使用确定性简单无向模型

### 决策

CS 核心第 18、19、20 课新建“可追踪图遍历实验”。公开图固定为零基顶点的简单无向无权图，构造时规范化端点并排序邻接表；BFS 在入队时标记并记录距离和父节点，DFS 按最小未访问顶点启动分量并通过父边识别无向环。

### 原因

方向、权重、平行边和自环会同时改变表示、距离与判环条件。先锁定简单无向无权模型，才能让邻接表与矩阵、`visited` 时机、父节点路径、连通分量和首次环证据形成确定性双语言测试；复杂图类型留到建立这个共同基线之后。

### 影响

- 无向边 `(u,v)` 规范化为 `(min,max)`，重复边和自环受控失败；邻接矩阵对称，邻接表条目总数为 `2|E|`。
- BFS 只对无权或所有边等代价的图给出最少边数；不可达距离和父节点保持为空。
- DFS 访问过的邻居只有在不是当前父节点时才构成无向环证据；全图扫描产生互不重叠的连通分量。
- 本轮不进入有向图、拓扑排序、强连通分量、带权图、Dijkstra、最小生成树、并查集或正式机考题。

## 2026-07-16: 树遍历使用双表示、显式递归保护和确定性前沿

### 决策

CS 核心第 15、16、17 课新建“可追踪树与遍历实验”。输入使用零基层序槽位并保留规范化副本，运行结构使用私有链接节点；递归遍历提供独立于运行时的显式深度保护，迭代 DFS 与 BFS 固定左右访问顺序并追踪最大前沿。

### 原因

槽位表示便于稳定构造和复现边界，链接节点便于解释所有权和真实遍历，两者并存可以明确区分接口与表示。显式深度上限允许安全观察递归失败而不制造栈溢出；确定性压栈和入队规则使双语言输出与操作证据可逐字比较。

### 影响

- 高度按边数定义：空树为 `-1`、根深度为 `0`、叶子高度为 `0`；槽位尾部空值裁剪，但非空输入的空根与孤儿节点被拒绝。
- C++ 树节点由 `std::unique_ptr` 单一拥有，树对象可移动不可复制；Python 与 C++ 均不向调用方暴露节点引用。
- 已验证树不需要 `visited`；一般图的环、重复边和访问集合留到图课程。
- 本轮不进入搜索树、平衡树、堆、优先队列、一般图、最短路、回溯、动态规划或正式机考题。

## 2026-07-16: 查找与排序使用不可变有序输入和迭代归并

### 决策

CS 核心第 12、13、14 课新建“可追踪查找与排序实验”。有序序列在 `SortedValues` 构造时复制并验证一次；基础排序与归并排序使用带标签元素显式验证稳定性；完整归并采用自底向上的迭代实现，不提前引入递归调用栈。

### 原因

把有序验证与查找计数分开，可以准确解释一次性前置成本和单次查询成本。带标签重复键能把“值已经排好”与“相等元素相对顺序保持”区分开。迭代归并则能先证明有序段、稳定合并与 `Theta(n log n)`，再把递归控制流留给后续专门课程。

### 影响

- `lower_bound` 与 `upper_bound` 统一维护 `[left, right)`，合法返回值允许等于序列长度。
- 插入排序只在严格越过时右移；选择排序保留真实交换导致的不稳定反例，所有排序均返回副本。
- 归并相等时固定先取左段，逐轮记录宽度、比较次数、写入次数和完整快照。
- 本轮不进入快速排序、堆排序、非比较排序、外部排序、答案二分、树、正式递归、并行排序或机考题。

## 2026-07-16: 哈希基础使用确定性教学模型并独立稳定输出

### 决策

CS 核心第 9、10、11 课新建“可追踪哈希实验”。教学哈希只接受非负整数键，固定为 `key % bucket_count`；哈希表使用分离链接、1.0 最大负载因子和倍增扩容。集合与频次报告通过显式保序或排序建立跨语言契约，不读取标准无序容器的内部桶序列。

### 原因

真实语言运行时与标准库允许不同哈希算法、桶策略和遍历顺序，直接把某一实现细节写成课程契约会造成错误迁移。确定性模型能让冲突链、键比较次数、负载边界和扩容搬移成为可测试证据，同时仍可准确说明标准容器只保证的抽象行为。

### 影响

- 相等键必须得到相同哈希，但同一哈希值不推出键相等；冲突由桶链中的键比较处理。
- 更新已有键不增加大小，也不触发预扩容；插入新键越过负载阈值时先搬移旧键，再插入新键。
- Python `set`／`dict` 和 C++ `unordered_set`／`unordered_map` 只用于接口对照与应用实现，报告不依赖其遍历顺序。
- 本轮不进入开放寻址、密码学哈希、哈希攻击、布隆过滤器、LRU、并发哈希表、排序、树、递归或面试题。

## 2026-07-16: 链表、栈和队列使用独立双语言阶段作品

### 决策

CS 核心第 6、7、8 课新建“可追踪线性结构实验”，不继续扩张“可追踪数组实验”。单链表公开精确访问次数；栈只在链表头修改；队列同时维护拥有头节点和非拥有尾指针。Python 空结构操作抛出 `IndexError`，C++ 抛出 `std::out_of_range`。

### 原因

数组实验围绕连续表示、索引和容量；线性结构实验围绕节点链接、单一所有权与首尾状态。独立成果线能冻结已有输出，也让学习者先理解节点所有权，再复用同一结构推导 LIFO 和 FIFO。

### 影响

- C++ 容器为 move-only，不定义含糊的浅复制语义，也不向调用方暴露节点指针。
- 队列尾指针只观察拥有链；删除最后一个节点时必须重置为空。
- 本轮不进入双向或循环链表、环形缓冲区、优先队列、哈希、排序、递归和面试题。

## 2026-07-16: CS 主干先用数组边界与操作计数建立双语言证据

### 决策

CS 核心首批正文拆为两课：第 1 课建立序列接口、数组表示、检查式访问和复制边界；第 2 课用线性查找、两两比较与增长表建立操作计数和渐近复杂度。两课共用新的 Python/C++“可追踪数组实验”，不继续扩张语言阶段的学习进度报告器。

### 原因

数组边界是后续链表、栈队列、排序和图结构的共同输入模型，复杂度又需要先固定输入规模与基本操作。把二者连续建设，能同时避免脱离代码背诵 Big O，也避免一次塞入过多数据结构。独立实验成果线可以保持学习进度报告器的语言工程化边界清晰。

### 影响

- Python `list` 只作为序列实现使用，不描述为传统同类型连续数组；C++ 使用 `std::vector` 与只读 `std::span`说明连续容器。
- Python 主动拒绝负索引；C++ 使用显式检查与 `at()`，不执行越界未定义行为。
- 操作次数是自动验收事实，真实计时只是辅助观察；不以单次计时判断复杂度或语言性能。
- 本轮不接入求职题，不读取 recruiting raw 素材，也不进入字符串、二维数组、摊还分析、链表、栈队列和排序。

- 配置优先级为命令行、显式 TOML、默认值；不自动读取用户目录或环境变量。运行依赖继续只有 Python 标准库。
- 成功、业务失败和参数语法错误分别使用退出码 0、1、2；日志只进入标准错误，主报告契约保持稳定。

## 2026-07-13: 语言主线采用长期能力阶梯

### 决策

### 影响

- 当前两篇Python起步正文保持不变，后续课程按更高完成标准继续建设。
- CS最小核心只要求核心语言和基础工程化验收，不要求先学完全部高级主题。
- 并发、网络、性能和领域能力在Web、AI、工业控制及系统项目中持续回链语言课程。

## 2026-06-25: Docker、算法和数据库采用分层学习口径

### 决策

- Docker进入工程基础，但语言前只要求最小认知：镜像、容器、端口、挂载、环境变量和常用查看命令。
- Dockerfile、docker compose、数据库容器、服务部署、监控和回滚在后续项目中逐步深化。
- 算法拆成两层：CS核心学习通用算法与复杂度；AI基础学习机器学习、深度学习、NLP/Transformer和强化学习等模型算法。
- 数据库实践优先使用 PostgreSQL 和 `psql`；MySQL作为常见生产数据库对照；SQLite只用于轻量本地练习或特定样例。

### 原因

Docker过早深入会拖慢小白进入编程，但后续部署和复现离不开容器认知。算法如果只写一个词，容易混淆刷题、数据结构和模型训练。数据库如果只写SQL，也会忽略真实工程中连接、迁移、事务、索引和部署工具链的差异。

### 影响

- 工程基础课程需要加入Docker最小认知章节，但不把容器部署设为语言前门槛。
- CS核心课程必须覆盖通用算法，不把机器学习内容提前塞进CS阶段。
- AI基础课程必须明确模型算法的前置和边界，NLP/Transformer作为LLM/Agent关键桥梁。
- Web与CS数据库内容默认围绕PostgreSQL/psql组织，再补MySQL差异。
