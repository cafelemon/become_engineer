# 决策记录

## 2026-06-25: 仓库采用公开学习平台定位

### 决策

本仓库定位为公开学习平台，覆盖工程开发、数据库、AI、LLM 和 Agent 等方向。

### 原因

项目既服务个人长期学习，也希望形成面向公众可阅读、可复用、可纠错的知识沉淀。

### 影响

- 文档需要面向外部读者，而不是只写给自己。
- 资源收集需要保留来源和质量判断。
- 笔记需要从个人草稿整理为公开表达。

## 2026-06-25: 个人学习资料不进入公开仓库

### 决策

本地 `个人学习/` 目录作为私有资料区，加入 `.gitignore`，不提交到 GitHub。

### 原因

个人学习资料可能包含未整理草稿、下载资料、隐私信息和版权风险内容，不适合公开。

### 影响

- 公开内容需要从私有目录中筛选、整理后再进入仓库。
- 提交前必须检查 `git status`，确认私有目录未被跟踪。

## 2026-06-25: 初期以 Markdown 仓库为主

### 决策

项目初期不急于搭建复杂站点，先使用 Markdown 和目录结构沉淀内容。

### 原因

当前最重要的是建立内容体系、筛选标准和长期维护节奏。

### 影响

- 后续可以平滑迁移到 VitePress、Docusaurus 或 MkDocs。
- 现阶段优先完善内容结构和质量标准。

## 2026-06-25: 采用四层六线结构

### 决策

项目采用四层六线结构组织公开内容。

四层为学习地图、学习路线、项目实践、资源评测。六线为工程基础、编程语言、Web 全栈、CS 核心、AI 基础、LLM/Agent。

### 原因

单纯资源列表容易堆链接，单纯课程路线又难以承载个人学习沉淀。四层六线可以同时保留全局导航、阶段路线、实践验证和资源判断。

### 影响

- 新增内容必须先映射到主线和内容层级。
- `resources/` 记录资源事实，`reviews/` 记录资源判断。
- 从 `个人学习/` 提炼的内容优先进入 `publications/`、`notes/`、`projects/` 或 `reviews/`。
