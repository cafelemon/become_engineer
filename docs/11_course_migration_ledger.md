# 正式课程 V2 迁移台账

## 台账用途

这份台账先把 55 节既有课程从“已经有内容”推进到“符合新架构、能连续学习”，现在继续登记按 V2 新建的正式课程。它不替代公开课程地图；公开层级和课程入口仍以 `learning-paths/curriculum-map.md` 为准，机器可校验的逐课记录位于 `site-src/data/curriculum/migration-v2.json`。

当前结论：七个连续批次、55 节既有课程已经全部迁移为 V2；第八批 Web 起步四课也已完成，第九批 Web 核心已开放前三课，目前正式课程共 62 节。学习进度报告器 Web 从 HTML 演进到带运行时契约、SQLite 持久化、REST CRUD、游标分页与重复写入保护的 v0.7。既有课程 URL 和稳定 `tutor lesson_id` 未改变。

## 目录重分层结论

| 当前课程组 | 目标层级 | 处理方式 |
| --- | --- | --- |
| 工程基础 10 节 | 工程基础起步 | 继续作为共同基座，用同一个工程学习工作台串联 |
| Python 起步 7 节 | Python 编程起步 | 保持顺序，用学习进度报告器形成第一个阶段作品 |
| CS 前 4 节 | 真正的 CS 起步 | 降低进入门槛，从 Python 中已经见过的数据、位置和操作成本开始 |
| Python 核心前 5 节 | Python 核心 | 保留深度，重组为理解、例子、复现、修改、排错和项目演进 |
| Python 核心后 2 节 | Python 工程化 | 强调包、CLI、配置、日志、诊断和工程回归 |
| C++ 当前前 2 节 | C++ 起步 | URL 不变，改为编译、类型、输入输出、函数和第一次程序组织 |
| C++ 当前后 3 节 | C++ 核心 | 进入 CMake、STL、对象、生命周期与 RAII |
| CS 第 5–20 节 | 共同算法与数据结构基础 | 使用人工小数据、轨迹、操作计数和边界测试 |
| CS 第 21–26 节 | 算法核心 | 保留堆、最短路、并查集和最小生成森林的知识深度 |

这次重分层解决两个主要断点：C++ 不再从“核心语言”突然开始；CS 前四课不再默认学习者已经能读底层实现。现有文件名和 URL 可以继续使用，公开标题只在内容迁移时按实际教学范围修订。

## 已有样板如何使用

现有六节对应样板均已吸收到正式课程，评审路径继续作为冻结回归证据，不进入生产导航或搜索。后续课程按同一规范生产，但必须根据课型选择合适的图解、复现和失败方式。

## 四类用户如何使用同一批课程

- 小白兴趣：从批次 1 开始，完成共同基座后按兴趣进入方向。
- 小白求职：同样完成共同基座，在阶段作品之后叠加原创机考、项目证据问答和复盘。
- 有基础兴趣：通过能力测评跳过已经证明掌握的课程，从具体缺口或方向核心进入。
- 有基础求职：补缺口后进入方向核心与深化，同时叠加求职训练。

四类用户不复制四套正文。新手操作、深化挑战和求职训练作为可组合内容进入同一个课程与项目体系。

## 状态更新规则

机器台账中的状态只允许：

- `待迁移`：只有 V1 正式内容。
- `样板已通过待迁移`：有评审样板，但正式页面仍是 V1。
- `迁移中`：页面、知识库、测试和项目正在同一批次修改，尚未对外宣称完成。
- `已迁移`：页面和知识库均为 V2，示例、检索、构建、移动端和项目回归全部通过。

每次修改台账后执行：

```bash
node scripts/validate_curriculum_v2.cjs
node scripts/test_tutor_search.cjs
node scripts/validate_course_content_v2.cjs
mkdocs build --strict
git diff --check
```

当前统计：9 个批次，62 节逐课登记；全部正式课程使用 V2。共同算法与数据结构基础完成 16/16，算法核心完成 6/6，Web 起步完成 4/4，Web 核心完成 3/4。既有课程的剩余验收门是四类用户真实试学；新课继续逐节完成示例、助教、构建与画像规则检查。
