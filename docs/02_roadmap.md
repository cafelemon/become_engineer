- 设计公开内容目录结构。

- 建立六条主线：工程基础、编程语言、Web 全栈、CS 核心、AI 基础、LLM/Agent。
- 为每条主线建立入口页。
- 设计资源条目模板。
- 设计学习笔记模板。
- 设计练习与项目模板。
- 设计资源评测模板和个人学习转公开输出模板。

## V0.3 首批资源沉淀

- 为六条主线建立首批资源清单。
- 对资源进行基础质量评估。
- 标记推荐学习顺序。
- 区分资源事实记录和资源评测判断。
- 用一个小主题验证外部素材从抓取、清洗、纠错、重写到实践入库的完整流程。
- 完成《大模型全部学习笔记》2519 页的逐项分类、覆盖矩阵和缺口分析。
- 明确大模型笔记主要服务Python、AI基础和LLM/Agent；其他主线的外部来源已登记，课程按统一路线逐阶段加工。

## V0.4 首批学习路径

- 建立面向小白的唯一默认学习顺序。
- 建立工程基础前置路径。
- 建立Python起步与Python/C++双主修路径。
- 将语言学习分为入门、进阶、工程化、深入和领域应用，持续贯穿后续项目。
- 建立Java补充路线。
- 建立CS最小核心，再进入Web与CS深化并行。
- 在 CS 深化中登记可跳过的工业控制与实时通信能力，后续扩展为“设备系统与边缘智能”跨主线专业领域。
- 建立AI基础与LLM/Agent后置路径。
- 建立完整课程地图，将CS、Web、AI和LLM/Agent统一拆为必修主干、推荐深化和专业选修。
- 为推荐深化和专业选修建立用途、前置、时机、实践、验收、跳过影响和素材状态契约。
- 将来源状态与公开课程完成状态分开记录。

## V0.5 实践项目

- 建立跨笔记、跨知识模块的项目主干。
- 建立聚焦实验和阶段练习。
- 补充验收标准和复盘模板。
- 将个人学习成果中适合公开的部分整理进 `publications/`、`notes/`、`projects/` 或 `reviews/`。
- 已规划 Python 内容分析、结构化数据 ML、文本分类演进、强化学习控制、智能学习助手五个项目。
- 已完成离线可测的 Tool Calling 安全销售查询练习。

## V0.6 顺序化课程建设

- 按统一路线逐阶段建设，不同时铺开所有主线。
- 第一批正文从工程基础前置开始。
- 每个单元包含前置、步骤、示例、练习、错误排查、验收和下一步。
- 项目只在达到阶段解锁条件后出现。
- 设备系统专业领域先建设仿真课程与集成验证，再规划真实设备联调；本阶段不创建空项目目录。
- CS算法机考先登记为推荐深化练习体系，不建立独立项目或提前编写题库正文。
- 后续课程建设从必修主干开始，再按前置逐项开放推荐深化和专业选修。
- 建立统一课程内容规范和单节课程模板，覆盖AI协作、视觉呈现、可复现代码、手动检查点和三级实践。
- 新增课程从创建时执行内容规范；已有课程只在后续扩写或重构时迁移，不为统一格式批量返工。
- 默认由连续3-6节相关课程形成阶段作品，优先接入 `exercises/` 或既有长期项目里程碑。
