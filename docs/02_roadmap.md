# 版本规划

## V0.1 项目初始化

- 建立公开仓库和远程连接。
- 建立 README 与 docs 项目文档。
- 明确公开内容与个人私有内容边界。
- 配置 `.gitignore` 排除 `个人学习/`。
- 建立第一版学习领域清单。

## V0.2 内容结构

- 设计公开内容目录结构。
- 为每个学习领域建立入口页。
- 设计资源条目模板。
- 设计学习笔记模板。
- 设计练习与项目模板。

## V0.3 首批资源沉淀

- 为 Python、前端、后端、数据库建立首批资源清单。
- 为机器学习、深度学习、LLM、Agent 建立首批资源清单。
- 对资源进行基础质量评估。
- 标记推荐学习顺序。

## V0.4 首批学习路径

- 建立 Python 到后端实践路径。
- 建立前端到全栈实践路径。
- 建立机器学习到深度学习路径。
- 建立 LLM 应用与 Agent 入门路径。

## V0.5 实践项目

- 设计小型练习项目。
- 设计综合项目。
- 补充验收标准和复盘模板。
- 将个人学习成果中适合公开的部分整理进公开内容区。

## V1.0 公开可用版本

- 形成稳定目录结构。
- 每个主方向至少有一条可执行学习路线。
- 每条路线至少包含资源、笔记、练习和项目建议。
- 建立贡献与纠错说明。
- 完成一次完整内容审查。
