# 差异与后续修订

## 正式保留

| 已验证做法 | 为什么保留 |
| --- | --- |
| 结果、问题、界面或现象开场 | 新手先知道要解决什么，再进入操作 |
| 七类共同语义环节 | 十三个样板都能自然覆盖，又不要求固定步骤 |
| 微型例子、主动修改和错误恢复 | 能形成复现、理解和独立操作的闭环 |
| 功能型视觉与静态降级 | 图和交互帮助理解，JavaScript 失效也能继续学习 |
| 连续作品线 | 把多节课程接成可运行、可复盘、可求职表达的成果 |
| 离线优先、真实服务可选 | 降低费用、网络、密钥和平台波动对学习的影响 |
| 小码的分层提示和不猜测降级 | 给帮助但不代替学习者完成判断 |

## 按课型变化

| 不应该统一的内容 | 采用规则 |
| --- | --- |
| 第一屏视觉 | 工具看界面，算法看状态，系统看生命周期或时序，AI 看数据与结果 |
| 例子与运行方式 | Python 微例可进浏览器；C、C++、服务和设备保留本地工具链 |
| 失败设计 | 算法观察错误状态，系统避免执行未定义行为，AI／LLM 覆盖不可信输入 |
| 深化内容 | 只在原理、性能、复杂度、硬件或职业方向确实需要时出现 |
| 项目关系 | 接入六条长期作品线，算法使用阶段实验，不创建覆盖全站的超级项目 |

## 小码实现差异

三批知识库 Schema 已经一致，差异主要在浏览器运行时：上下文观察方式、欢迎语、存储键、焦点处理和代码组织略有不同。本统一站点使用一份隔离候选运行时，取以下共同结果：

- 当前语义区域决定推荐问题。
- 大面积桌面面板与手机全屏布局。
- 键盘可以打开、提问、展开和关闭。
- 关闭后焦点回到入口。
- `localStorage` 不保存原始提问。

统一评审站点现在通过薄适配器使用正式双协议运行时；样板知识库仍保持冻结。
