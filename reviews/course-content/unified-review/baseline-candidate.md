# 候选统一基线

## 统一学习闭环，不统一页面模样

十三个样板已经证明，课程可以共享一组学习环节，同时根据内容调整顺序、次数和视觉。正式规范应描述“学习者必须得到什么”，而不是要求作者复制固定章节。

| 语义环节 | 学习者在这里解决什么 | 是否必需 |
| --- | --- | --- |
| `overview` | 先看结果、问题、界面或现象，知道为什么值得学 | 必需 |
| `concept` | 把关键关系想清楚，建立足够开始实践的理解 | 必需，可重复 |
| `example` | 用足够短的例子观察一件事 | 必需，可重复 |
| `reproduce` | 按明确环境、命令和预期结果复现 | 必需，可重复 |
| `modify` | 改成自己的数据、规则或行为 | 必需 |
| `troubleshoot` | 遇到典型失败，知道从哪里查和怎样恢复 | 必需 |
| `project` | 说明本课进入哪条作品线，留下什么结果 | 必需；无法接入时说明替代验收 |
| `deepen` | 进入实现、性能、复杂度或硬件边界 | 可选 |
| `career` | 把代码、测试和复盘组织成求职表达 | 可选 |

课程不固定环节数量，也不要求按表格顺序排列。第一屏只需先给方向感，避免学习者还没理解问题就连续执行多个任务。

## 八类课各自保留什么

| 课型 | 开场 | 核心组织 | 必须看见的证据 |
| --- | --- | --- | --- |
| 工具操作 | 完成后的界面 | 找入口、操作、确认变化、走错恢复 | 编号截图、文件或命令结果 |
| 编程起步 | 几行代码的输出 | 预测、运行、修改、读错误 | 代码、输出、项目版本变化 |
| CS 概念 | 程序为何这样运行 | 具体对象、表示、状态、抽象和边界 | 结构图、状态表、实现对照 |
| 算法与数据结构 | 先人工解决小问题 | 建模、轨迹、规律、实现和复杂度 | 可回放轨迹、边界测试和计数 |
| 系统与设备 | 故障、资源或时序现象 | 边界、最小实验、机制、调试和可靠性 | 生命周期、时序、日志、仿真／真机说明 |
| AI 实验 | 数据、目标和基线结果 | 划分、基线、训练、验证、误差和复现 | 配置、指标、错误样本和结果对照 |
| Web／LLM／Agent | 请求、失败响应或受限流程 | 契约、调用链、错误边界、评估和恢复 | 请求响应、状态、权限和离线替代 |
| 项目整合 | 当前版本和新增需求 | 设计、增量实现、测试、演示和复盘 | 版本差异、测试报告和求职叙事 |

## 功能型视觉与交互

- 每张图只回答一个问题，并配等价文字说明。
- 状态变化难以静态表达时才使用播放器；必须支持前进、后退、重置和静态表降级。
- 浏览器运行器只承载安全的微型代码；正式项目仍在本地终端运行。
- C、C++、服务和设备课程使用真实本地工具链；设备课程继续坚持“仿真入门、真机毕业”。
- 外部模型、硬件和付费服务必须有离线、Mock 或低成本路径，失败时课程正文仍能继续。
- 所有交互支持键盘、390px、深浅色和减少动画；颜色不是唯一状态信息。

## 小码 V2 候选协议

十三套知识库已经使用同一结构：

```text
lesson
contexts[]  -> id / type / title / anchor
cards[]     -> context_id / question / aliases / keywords
               diagnostic / hints / example / answer / source
```

候选运行时遵守以下规则：

- 用当前阅读区域的 `context_id` 提升相关问题排序。
- 每张卡先问诊断问题，再逐层展开两条提示、局部例子和答案。
- 搜索返回最多三项；低于阈值时明确说没有可靠答案。
- 只保存面板开关与提示层级，不保存原始提问。
- 桌面使用大面积侧边聊天窗，手机使用全屏面板，关闭后焦点回到入口。
- 正式升级时同时读取旧 `steps` 与新 `contexts`，直到 55 节课迁移完毕。

## 课程怎样进入项目

每节进入连续作品线的课程都要回答五个问题：

1. 项目上一版已经会做什么？
2. 这节课新增什么可以运行或观察的能力？
3. 哪些文件、接口或数据发生了变化？
4. 学习者需要保存什么运行、测试、排错或解释结果？
5. 下一节准备继续加什么？

项目关系是内容的一部分，不应只在课程结尾挂一个链接。纯理论课无法自然接入项目时，可以使用推理记录、实验结果或概念判断作为替代，但要说明原因。

## 语言基线

- 像老师坐在旁边讲，不写评审报告式正文。
- 技术定义短而准确，操作说明交代位置、变化和恢复方式。
- 少用整齐排比、口号和“可观察结果、成功证据、迁移验收”等管理化词语。
- 可以说“这里先别急着”“我更建议”，但不虚构个人经历和行业故事。
