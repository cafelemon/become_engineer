# Become Engineer 总地图

## 四层结构

```text
学习地图
  -> 学习路线
    -> 项目实践
      -> 资源评测与公开沉淀
```

## 六条主线

```text
工程基础
  -> 编程语言
    -> Web 全栈
      -> CS 核心
        -> AI 基础
          -> LLM/Agent
```

这不是严格的单行顺序，而是推荐的能力依赖：工程基础和编程语言越扎实，后面的 Web、CS、AI 和 Agent 学习越容易形成闭环。

## 推荐学习节奏

1. 先建立工程基础：会用 Git、命令行、编辑器、调试、测试和文档。
2. 选择主语言：Python 用于自动化、数据和 AI；C++ 用于底层、性能和系统理解；JavaScript/TypeScript 用于 Web。
3. 做 Web 全栈项目：把前端、后端、数据库和部署串起来。
4. 补 CS 核心：用算法、网络、操作系统和系统设计解释真实工程问题。
5. 进入 AI 基础：理解机器学习、深度学习和强化学习的基本范式。
6. 进入 LLM/Agent：围绕 RAG、工具调用、评估和 Agent 项目做应用实践。

## 内容沉淀规则

- 路线放入 `learning-paths/`。
- 原始免费资源放入 `resources/`。
- 资源使用后的判断放入 `reviews/`。
- 可公开知识笔记放入 `notes/`。
- 练习题和实验放入 `exercises/`。
- 完整实践项目放入 `projects/`。
- 从 `个人学习/` 提炼出的阶段成果放入 `publications/`。
