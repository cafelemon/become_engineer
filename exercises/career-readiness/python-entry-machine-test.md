# Python 起步机考与限时模拟

这是通用软件工程求职路线的第一个训练包。它不替代 Python 课程，也不要求兴趣路线完成；完成 Python 起步和学习进度报告器基础版本后，用它检验“读取题意、处理输入、验证边界、在限制内交付”的能力。

所有题目均为本项目围绕学习记录场景独立编写，不冒充任何公司的真实题库。企业或平台题目只有来源和授权明确后才会单独登记。

## 使用规则

1. 每道题新建独立目录，至少包含 `solution.py`、`README.md` 和测试记录。
2. 先手写输入、输出和边界，再请求 AI 给提示；不要先复制完整答案。
3. 程序只使用 Python 3.11 标准库，从标准输入读取，从标准输出返回结果。
4. 输出必须与题目一致；调试信息写到标准错误或在提交前删除。
5. 每题记录开始、首次通过、最终完成三个时间点和失败原因；时间用于复盘，不用于和别人比较。

## Level 1：基础实现

### 题 1：规范化课程名称

输入第一行是课程数量，后续每行是一个可能包含首尾空格的课程名。去掉首尾空格，忽略空行，保持首次出现顺序并去重。

输入：

```text
6
 Python 起步
Git
Python 起步

 C++ 核心 
Git
```

输出：

```text
Python 起步
Git
C++ 核心
```

手动检查点：解释为什么不能只用 `set` 直接输出，以及空字符串如何处理。

### 题 2：判断学习状态

每行给出课程名、计划小时、完成小时，以 `|` 分隔。按规则输出状态：完成小时为 0 是“未开始”；达到或超过计划是“已完成”；其余是“进行中”。计划小时必须大于 0，非法行输出 `INVALID`。

```text
Python|10|7
Git|6|6
算法|0|0
```

```text
Python|进行中
Git|已完成
算法|INVALID
```

失败实验：加入 `测试|5|eight`，记录类型转换错误发生在哪里，再改为可控的非法输入处理。

## Level 2：数据组织

### 题 3：标签频次

每行包含一个用逗号分隔的标签列表。忽略空标签，同一行重复标签只计一次，最后按“出现次数降序、标签名字典序升序”输出。

```text
python,基础,基础
git,工程
python,工程
```

```text
工程 2
python 2
git 1
基础 1
```

手动检查点：说明“同一行去重”和“全部行累计”分别适合用什么容器。

### 题 4：稳定排序学习记录

输入若干 `课程名|计划小时|完成小时`，按进度百分比降序排列；进度相同时保持原输入顺序。进度超过 100% 时按实际比例计算，不截断。

验收至少覆盖：空输入、相同进度、小数小时、超额完成和中文课程名。

## Level 3：混合题

### 题 5：生成可筛选摘要

组合状态、标签与排序：只输出带有指定标签的课程，先按状态“进行中、未开始、已完成”排序，同状态再按课程名排序，并在最后输出匹配数量。

要求把解析、校验、筛选、排序和格式化拆成独立函数；至少一个函数必须有直接单元测试。

### 题 6：修复一份错误实现

让 AI 或同伴故意生成一份包含以下三类问题的版本：把字符串直接参与加法、在遍历列表时删除元素、对空输入调用 `max()`。你需要：

- 用最小输入复现每个错误。
- 写下根因而不是只粘贴报错。
- 每修复一类问题补一个回归测试。
- 解释为什么最终测试能阻止同类错误再次出现。

## 限时模拟：学习进度汇总

### 约束

- 建议限时 45 分钟；如果未完成，停止编码并保留现场，先复盘再继续。
- 不使用第三方库，不读取固定绝对路径，不修改输入数据。
- 程序名为 `solution.py`，使用 `python solution.py < input.txt` 运行。

### 输入

第一行是记录数量 `N`。后续 `N` 行格式为：

```text
课程名|计划小时|完成小时|逗号分隔标签
```

固定样例：

```text
4
Python|10|7|python,基础
Git|6|6|git,工程
算法|8|4|算法,基础
测试|5|0|测试,工程
```

### 输出

```text
已完成: 1
进行中: 2
未开始: 1
总计划: 29.0
总完成: 17.0
最高进度: Git 100.0%
热门标签: 基础, 工程
```

热门标签取出现次数最高的全部标签，按字典序输出。最高进度相同时选择输入中先出现的课程。

### 必测边界

- `N = 0`。
- 计划小时为 0、完成小时为负数或字段数量不对。
- 完成小时超过计划小时。
- 同一课程重复标签。
- 两门课程进度相同。
- 中文名称和 UTF-8 输入。

## 提交证据

- 运行命令、固定输入和逐字输出。
- 至少 6 个自动测试，包含正常、边界和非法输入。
- 一次未通过的记录：输入、实际输出、预期输出、根因和修复。
- 一份复盘：哪一步耗时最多，下次怎样更早暴露问题。
- AI 协作记录：AI 提供了什么，哪一处由你否决或修改，如何验证。
