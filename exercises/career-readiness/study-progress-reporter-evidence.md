# 学习进度报告器项目证据问答

这个训练包把[双语言学习进度报告器](../programming-languages/study-progress-reporters/README.md)整理成可以核验的项目叙事。目标不是背一套标准话术，而是让每个回答都能回到代码、测试、命令、失败记录或设计取舍。

## 项目叙事骨架

用下面六项写一版不超过一页的项目说明：

1. **问题：** 零散学习记录为什么需要结构化报告器。
2. **约束：** Python 与 C++ 输出契约一致、输入只读、错误可观察、不依赖第三方库。
3. **演进：** 从单条记录到容器、模块、对象、资源和调用边界经历了哪些课程里程碑。
4. **关键决策：** 为什么选择数据类、迭代器、RAII、上下文管理器、测试和稳定输出契约。
5. **失败与修复：** 至少选择一次真实失败，说明如何复现、定位、修改和防回归。
6. **证据：** 给出实际运行命令、测试数量、代表性输入输出和 Python/C++ 一致性结果。

不要虚构用户量、性能提升或线上规模。没有测量的数据就写“尚未测量”，把可重复验证的事实说清楚。

## 问答规则

- 先独立回答，再展开课程或请小码提示。
- 每个回答必须引用至少一种证据：函数/类型名、测试用例、命令输出、数据契约或失败日志。
- 回答“为什么”时同时说明替代方案和当前取舍，不只解释语法。
- 不知道时明确指出缺口，并链接回要补的课程；不要编造项目中不存在的实现。

## 第一组：问题与边界

### 1. 这个项目解决了什么问题，为什么不是普通练习脚本？

证据要求：指出多条学习记录、稳定报告、外部输入、错误退出和双语言实现中的至少三项。

### 2. 为什么 Python 与 C++ 要保持逐字一致的主报告？

证据要求：解释共同数据/行为契约，以及一致性测试能发现什么、不能证明什么。

### 3. 哪些内容故意没有进入主报告标准输出？

证据要求：说明调用追踪、审计文件和错误信息各自的输出边界，解释避免隐式日志的理由。

## 第二组：数据与算法

### 4. 为什么从 `TypedDict` 迁移到 `@dataclass`？

证据要求：列举一个对象方法、复制行为和 `default_factory` 的受控失败实验；同时说明字典仍然适合什么边界。

### 5. 为什么惰性遍历需要在报告边界显式物化？

证据要求：说明生成器一次消费问题，以及汇总、排序或多次遍历怎样影响选择。

### 6. 排序、筛选和汇总如何保证不修改调用者输入？

证据要求：指出相应测试或复制边界，给出如果原地排序会造成的可观察回归。

## 第三组：资源与失败

### 7. C++ RAII 与 Python `with` 在这个项目中解决了什么共同问题？

证据要求：引用审计文件打开、正常退出和失败路径；不要用垃圾回收时机代替资源边界证明。

### 8. 为什么 Python 后续又增加了临时文件后再 `replace()`？

证据要求：说明直接覆盖的风险、块内异常时旧文件为何保留、临时文件怎样清理。

### 9. `write_audit_snapshot()` 返回布尔值有什么优点和限制？

证据要求：说明与 C++ 简单契约的对照，以及真实生产系统为什么可能需要更详细错误类型或日志。

## 第四组：测试与交付

### 10. 你怎样证明“功能完成”，而不只展示一张截图？

证据要求：运行 mypy、Python unittest、CMake/CTest 和跨语言报告比较中的至少三项，并解释各自覆盖的风险。

### 11. 选一个真实失败，完整说明诊断过程。

回答结构固定为：最小复现 → 实际与预期 → 假设 → 检查 → 修改 → 回归测试。没有实际失败时，先完成课程中的受控失败实验。

### 12. 如果下一步把它做成 CLI，当前结构哪里会变化，哪里不应该变化？

证据要求：区分领域模型、报告逻辑、文件资源、入口参数、配置、日志和退出码，不要把 CLI 解析写进核心报告函数。

## 三分钟口头版本

按下面顺序录制或向同伴讲一次，不看稿也可以回到证据：

1. 30 秒：问题与目标用户。
2. 45 秒：从 Python 起步到双语言阶段作品的演进。
3. 60 秒：一个关键技术决策和一个失败修复。
4. 30 秒：测试与可复现证据。
5. 15 秒：当前边界和下一步。

时间限制用于训练信息取舍，不要求把所有细节塞进三分钟。被追问时再展开对应代码和测试。

## 验收

- 一页项目说明没有虚构指标，每项事实都能定位证据。
- 12 个问题至少完成 8 个，其中必须包含失败诊断、资源管理和测试交付。
- 同伴随机追问两个答案时，能打开对应文件或运行命令验证。
- 至少一次主动修订：删掉空泛术语，换成项目中的具体行为与取舍。
- 能明确说出当前项目不是生产级平台，下一步是包结构、CLI、配置与日志。
